# **[[2.3 Ternary Operator]]**

## Overview

This topic covers the ternary operator for inline conditional expressions. You'll learn the condition ? true : false syntax, nested ternary expressions, appropriate use cases, readability considerations, and shorthand conditional patterns.

## Condition ? True : False

The ternary operator is a compact alternative to if-else for simple assignments.

**Syntax:**
```javascript
condition ? expressionIfTrue : expressionIfFalse
```

**Basic Examples:**
```javascript
let age = 20;
let status = age >= 18 ? "Adult" : "Minor";
console.log(status);  // "Adult"

let score = 75;
let result = score >= 60 ? "Pass" : "Fail";
console.log(result);  // "Pass"

let hour = 14;
let greeting = hour < 12 ? "Good morning" : "Good afternoon";
console.log(greeting);  // "Good afternoon"
```

**Comparison with If-Else:**
```javascript
// Ternary operator
let access = isLoggedIn ? "Welcome" : "Please log in";

// Equivalent if-else
let access;
if (isLoggedIn) {
    access = "Welcome";
} else {
    access = "Please log in";
}
```

**With Function Calls:**
```javascript
let isValid = true;

// Execute different functions based on condition
isValid ? processData() : showError();

// Return different function results
let result = isValid ? calculateTotal() : getDefaultValue();
```

## Nested Ternary

Multiple ternary operators can be chained for multiple conditions.

**Syntax:**
```javascript
condition1 ? value1 : condition2 ? value2 : value3
```

**Grade Example:**
```javascript
let score = 85;

let grade = score >= 90 ? "A" :
            score >= 80 ? "B" :
            score >= 70 ? "C" :
            score >= 60 ? "D" : "F";

console.log(grade);  // "B"
```

**Multiple Conditions:**
```javascript
let age = 25;

let category = age < 13 ? "child" :
               age < 20 ? "teenager" :
               age < 60 ? "adult" : "senior";

console.log(category);  // "adult"
```

**Time-Based Message:**
```javascript
let hour = 15;

let timeOfDay = hour < 6 ? "night" :
                hour < 12 ? "morning" :
                hour < 18 ? "afternoon" : "evening";

console.log("Good " + timeOfDay);  // "Good afternoon"
```

**Warning:** Deeply nested ternaries become hard to read:
```javascript
// Avoid: Too complex
let result = a ? b ? c ? d : e : f : g;

// Better: Use if-else or extract to function
function getResult(a, b, c) {
    if (!a) return "g";
    if (!b) return "f";
    if (!c) return "e";
    return "d";
}
```

## Use Cases

The ternary operator excels in specific scenarios.

**Variable Assignment:**
```javascript
let theme = isDarkMode ? "dark" : "light";
let fontSize = isLargeText ? "18px" : "14px";
let currency = isEuropean ? "EUR" : "USD";
```

**Default Values:**
```javascript
let username = inputName ? inputName : "Guest";
// Or use || operator
let username = inputName || "Guest";
```

**Template Literals:**
```javascript
let user = { name: "John", isPremium: true };

let message = `Welcome, ${user.name}! ${user.isPremium ? "⭐ Premium member" : "Free tier"}`;
console.log(message);  // "Welcome, John! ⭐ Premium member"
```

**Object Properties:**
```javascript
let config = {
    theme: userPref ? userPref : "default",
    language: navigator.language ? navigator.language : "en"
};
```

**JSX/React Rendering:**
```javascript
// Common in React
function Greeting({ isLoggedIn }) {
    return (
        <div>
            {isLoggedIn ? <UserDashboard /> : <LoginButton />}
        </div>
    );
}
```

**Array Expressions:**
```javascript
let items = [
    { name: "Item 1", active: true },
    { name: "Item 2", active: false }
];

items.forEach(item => {
    console.log(`${item.name}: ${item.active ? "Active" : "Inactive"}`);
});
```

## Readability Considerations

Balance conciseness with clarity.

**Good: Simple, Clear Conditions**
```javascript
// Easy to understand
let status = isOnline ? "Online" : "Offline";
let icon = isLoading ? "spinner" : "checkmark";
let buttonText = isSubmitting ? "Saving..." : "Save";
```

**Avoid: Complex Expressions**
```javascript
// Hard to read
let result = user && user.permissions && user.permissions.admin ? 
    getAdminDashboard(user.id, true, config.adminOptions) : 
    getNormalDashboard(user.id, false, config.defaultOptions);

// Better: Use if-else
let result;
if (user?.permissions?.admin) {
    result = getAdminDashboard(user.id, true, config.adminOptions);
} else {
    result = getNormalDashboard(user.id, false, config.defaultOptions);
}
```

**Formatting for Readability:**
```javascript
// Multi-line formatting for nested ternaries
let discount = 
    customerType === "premium" ? 0.20 :
    customerType === "regular" ? 0.10 :
    customerType === "new" ? 0.05 : 0;
```

**When to Use If-Else Instead:**
```javascript
// Multiple statements needed
if (isValid) {
    logAction();
    processData();
    sendNotification();
} else {
    logError();
    showWarning();
}

// Complex side effects
if (condition) {
    // Multiple lines of logic
} else {
    // Multiple lines of alternative logic
}
```

## Shorthand Conditionals

Related patterns for conditional expressions.

**Logical AND (&&) for Conditional Execution:**
```javascript
// Execute only if condition is true
isLoggedIn && showDashboard();
hasPermission && deleteItem();

// Conditional property access
let name = user && user.name;  // undefined if user is falsy
```

**Logical OR (||) for Default Values:**
```javascript
let username = inputName || "Guest";
let port = config.port || 3000;
let timeout = options.timeout || 5000;
```

**Nullish Coalescing (??) for Null/Undefined Only:**
```javascript
// Only falls back for null or undefined
let count = userCount ?? 0;  // 0 is preserved, unlike ||

// Difference from ||
let value1 = 0 || 100;   // 100 (0 is falsy)
let value2 = 0 ?? 100;   // 0 (0 is not null/undefined)
```

**Optional Chaining (?.):**
```javascript
// Safe property access
let city = user?.address?.city;  // undefined if any part is null

// Combined with nullish coalescing
let city = user?.address?.city ?? "Unknown";
```

**Pattern Comparison:**
```javascript
// Ternary: two explicit outcomes
let status = isActive ? "active" : "inactive";

// && : conditional execution (no alternative)
isActive && console.log("Active");

// || : default value
let name = inputName || "Default";

// ?? : null-safe default
let count = nullableCount ?? 0;
```

## Summary / Key Takeaways

- **Ternary syntax:** `condition ? trueValue : falseValue`
- **Use for:** Simple, single-expression conditionals
- **Nested ternary:** Possible but use sparingly; format multi-line for readability
- **Best use cases:** Variable assignment, template literals, default values
- **Avoid for:** Complex logic, multiple statements, side effects
- **Related operators:** `&&` (conditional execution), `||` (fallback), `??` (null-safe default)
- **Readability first:** If ternary is hard to read, use if-else instead
- **Rule of thumb:** If you need more than one nested ternary, consider if-else

The ternary operator provides concise conditional expressions when used appropriately for simple decisions.
