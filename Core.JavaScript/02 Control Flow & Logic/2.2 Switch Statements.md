# **[[2.2 Switch Statements]]**

## Overview

This topic covers the switch statement for multi-way branching. You'll learn switch syntax, case statements for matching values, the break keyword for flow control, default cases for unmatched values, when to use switch vs if, and handling multiple conditions efficiently.

## Switch Syntax

The switch statement evaluates an expression and executes code based on matching cases.

**Basic Syntax:**
```javascript
switch (expression) {
    case value1:
        // Code for value1
        break;
    case value2:
        // Code for value2
        break;
    default:
        // Code when no case matches
}
```

**Simple Example:**
```javascript
let day = 3;
let dayName;

switch (day) {
    case 1:
        dayName = "Monday";
        break;
    case 2:
        dayName = "Tuesday";
        break;
    case 3:
        dayName = "Wednesday";
        break;
    case 4:
        dayName = "Thursday";
        break;
    case 5:
        dayName = "Friday";
        break;
    case 6:
        dayName = "Saturday";
        break;
    case 7:
        dayName = "Sunday";
        break;
    default:
        dayName = "Invalid day";
}

console.log(dayName);  // "Wednesday"
```

## Case Statements

Each case tests for equality with the switch expression.

**String Cases:**
```javascript
let fruit = "apple";

switch (fruit) {
    case "apple":
        console.log("Apples are $1.50/lb");
        break;
    case "banana":
        console.log("Bananas are $0.75/lb");
        break;
    case "orange":
        console.log("Oranges are $2.00/lb");
        break;
    default:
        console.log("Fruit not found");
}
```

**Strict Comparison:**
```javascript
let value = "5";

switch (value) {
    case 5:
        console.log("Number 5");
        break;
    case "5":
        console.log("String '5'");  // This matches
        break;
}
// Switch uses strict equality (===)
```

**Expression in Switch:**
```javascript
let x = 2;

switch (x * 2) {
    case 2:
        console.log("x is 1");
        break;
    case 4:
        console.log("x is 2");  // Matches: 2 * 2 = 4
        break;
    case 6:
        console.log("x is 3");
        break;
}
```

## Break

The `break` statement exits the switch block. Without it, execution "falls through" to the next case.

**With Break (Normal Behavior):**
```javascript
let color = "red";

switch (color) {
    case "red":
        console.log("Stop");
        break;  // Exits switch
    case "yellow":
        console.log("Caution");
        break;
    case "green":
        console.log("Go");
        break;
}
// Output: "Stop"
```

**Without Break (Fall-Through):**
```javascript
let grade = "B";

switch (grade) {
    case "A":
        console.log("Excellent");
    case "B":
        console.log("Good");
    case "C":
        console.log("Average");
        break;
    case "F":
        console.log("Fail");
}
// Output: "Good" then "Average" (falls through!)
```

**Intentional Fall-Through:**
```javascript
let month = 2;
let days;

switch (month) {
    case 1: case 3: case 5:
    case 7: case 8: case 10: case 12:
        days = 31;
        break;
    case 4: case 6: case 9: case 11:
        days = 30;
        break;
    case 2:
        days = 28;  // Simplified, ignoring leap years
        break;
}

console.log("Days in month:", days);
```

**Grouped Cases:**
```javascript
let day = "Saturday";

switch (day) {
    case "Monday":
    case "Tuesday":
    case "Wednesday":
    case "Thursday":
    case "Friday":
        console.log("Weekday");
        break;
    case "Saturday":
    case "Sunday":
        console.log("Weekend");
        break;
}
// Output: "Weekend"
```

## Default

The `default` case executes when no other case matches.

**Basic Default:**
```javascript
let status = "unknown";

switch (status) {
    case "active":
        console.log("Account is active");
        break;
    case "inactive":
        console.log("Account is inactive");
        break;
    case "suspended":
        console.log("Account is suspended");
        break;
    default:
        console.log("Unknown status");
}
// Output: "Unknown status"
```

**Default Position:**
```javascript
// Default can be anywhere (but conventionally at the end)
switch (value) {
    default:
        console.log("Default case");
        break;
    case 1:
        console.log("One");
        break;
    case 2:
        console.log("Two");
        break;
}
```

**No Default (Optional):**
```javascript
let code = 99;

switch (code) {
    case 1:
        console.log("Code 1");
        break;
    case 2:
        console.log("Code 2");
        break;
    // No default: nothing happens for unmatched values
}
```

## When to Use Switch vs If

**Use Switch When:**
```javascript
// Testing a single variable against multiple values
let action = "save";

switch (action) {
    case "save":
        saveFile();
        break;
    case "load":
        loadFile();
        break;
    case "delete":
        deleteFile();
        break;
}
```

**Use If-Else When:**
```javascript
// Range comparisons
let score = 85;

if (score >= 90) {
    grade = "A";
} else if (score >= 80) {
    grade = "B";
}

// Complex conditions
if (user.age >= 18 && user.hasLicense) {
    allowDriving();
}

// Non-discrete values
if (temperature > 30) {
    console.log("Hot");
}
```

**Comparison Table:**

| Scenario | Use Switch | Use If-Else |
|----------|------------|-------------|
| Exact value matching | ✓ | |
| Range comparisons | | ✓ |
| Complex boolean logic | | ✓ |
| Many discrete values | ✓ | |
| Two conditions only | | ✓ |

## Multiple Conditions

Handle multiple values mapping to the same code.

**Multiple Cases, Same Action:**
```javascript
let input = "Y";

switch (input.toUpperCase()) {
    case "Y":
    case "YES":
    case "YEAH":
    case "YEP":
        console.log("Confirmed");
        break;
    case "N":
    case "NO":
    case "NOPE":
        console.log("Cancelled");
        break;
    default:
        console.log("Please enter Yes or No");
}
```

**Command Handler:**
```javascript
let command = "help";

switch (command) {
    case "start":
    case "run":
    case "begin":
        startProcess();
        break;
    case "stop":
    case "end":
    case "quit":
    case "exit":
        stopProcess();
        break;
    case "help":
    case "?":
        showHelp();
        break;
    default:
        console.log("Unknown command");
}
```

**Return in Switch (Functions):**
```javascript
function getStatusMessage(code) {
    switch (code) {
        case 200:
            return "OK";
        case 201:
            return "Created";
        case 400:
            return "Bad Request";
        case 401:
            return "Unauthorized";
        case 404:
            return "Not Found";
        case 500:
            return "Internal Server Error";
        default:
            return "Unknown Status";
    }
}

console.log(getStatusMessage(404));  // "Not Found"
```

**Switch with Objects (Alternative):**
```javascript
// Object lookup can replace simple switches
const statusMessages = {
    200: "OK",
    201: "Created",
    400: "Bad Request",
    401: "Unauthorized",
    404: "Not Found",
    500: "Internal Server Error"
};

let message = statusMessages[404] || "Unknown Status";
console.log(message);  // "Not Found"
```

## Summary / Key Takeaways

- **Switch syntax:** `switch (expression) { case value: ... break; }`
- **Case statements:** Match values using strict equality (===)
- **Break keyword:** Required to prevent fall-through; exits the switch block
- **Default case:** Handles unmatched values; optional but recommended
- **Fall-through:** Without break, execution continues to next case; sometimes intentional for grouping
- **Use switch for:** Discrete value matching with many options
- **Use if-else for:** Ranges, complex conditions, boolean logic
- **Alternative:** Object lookup for simple value mappings

Switch statements provide clean, readable code when comparing a single value against multiple possibilities.
