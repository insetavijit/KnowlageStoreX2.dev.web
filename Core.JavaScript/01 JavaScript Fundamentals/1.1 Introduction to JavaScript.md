# **[[1.1 Introduction to JavaScript]]**

## Overview

This topic introduces JavaScript as the programming language of the web. You'll learn what JavaScript is, its history and evolution through ECMAScript standards, key differences between JavaScript and Java, common use cases, why JavaScript is essential for modern development, and the various execution environments where JavaScript runs.

## What is JavaScript?

JavaScript is a high-level, interpreted programming language that enables interactive web pages and dynamic user experiences. Originally designed to add interactivity to HTML pages, JavaScript has evolved into a versatile language that runs on browsers, servers, mobile devices, and desktop applications.

**Key Characteristics:**
- **Interpreted:** Executes directly without compilation
- **Dynamic typing:** Variables can hold any data type
- **Prototype-based:** Uses prototypes for inheritance
- **First-class functions:** Functions are treated as values
- **Event-driven:** Responds to user actions and system events

## History of JavaScript

**1995:** Created by Brendan Eich at Netscape in just 10 days, originally named Mocha, then LiveScript, finally JavaScript.

**1997:** ECMAScript standard established (ECMA-262) to ensure consistency across implementations.

**2009:** Node.js released, enabling JavaScript on servers.

**2015:** ES6 (ECMAScript 2015) introduced major language improvements including classes, modules, arrow functions, and promises.

**Present:** JavaScript is the most widely used programming language, with annual ECMAScript updates adding new features.

## ECMAScript Standards

ECMAScript is the specification that JavaScript implements. Each version adds new features:

- **ES5 (2009):** Strict mode, JSON support, array methods
- **ES6/ES2015:** Classes, modules, arrow functions, promises, let/const
- **ES2016+:** Annual releases with incremental improvements
- **ES2020:** Optional chaining, nullish coalescing
- **ES2021:** Logical assignment operators, numeric separators

Modern JavaScript developers write ES6+ code, which may be transpiled to ES5 for older browser support.

## JavaScript vs Java

Despite similar names, JavaScript and Java are fundamentally different:

| Aspect | JavaScript | Java |
|--------|-----------|------|
| **Type** | Scripting language | Programming language |
| **Typing** | Dynamic, weak | Static, strong |
| **Compilation** | Interpreted | Compiled to bytecode |
| **Execution** | Browser, Node.js | JVM (Java Virtual Machine) |
| **Object Model** | Prototype-based | Class-based |
| **Primary Use** | Web development | Enterprise applications |

The naming similarity is historical marketing, not technical relation.

## Execution Environments

JavaScript runs in various environments:

**Browser Environment:**
- V8 (Chrome, Edge, Brave)
- SpiderMonkey (Firefox)
- JavaScriptCore (Safari)
- Provides Web APIs (DOM, Fetch, Storage)

**Server Environment:**
- Node.js (V8-based JavaScript runtime)
- Deno (secure TypeScript/JavaScript runtime)
- Bun (fast all-in-one JavaScript runtime)
- Provides system APIs (File System, Network, Process)

**Embedded and IoT:**
- Espruino
- Johnny-Five (robotics)
- IoT.js

Each environment provides different APIs and capabilities, but core JavaScript syntax remains consistent.
