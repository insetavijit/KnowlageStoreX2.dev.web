# **[[1.7 Console & Output]]**

## Overview

This topic covers JavaScript's console object and output methods. You'll learn console.log for basic output, console.error and console.warn for different message types, console.table for structured data, console.time for performance measurement, debugging output techniques, and logging best practices.

## console.log

The most common method for outputting information during development.

**Basic Usage:**
```javascript
console.log("Hello, World!");
console.log(42);
console.log(true);

// Multiple arguments
console.log("Name:", "John", "Age:", 30);
// Output: Name: John Age: 30
```

**Variables and Expressions:**
```javascript
let name = "Alice";
let age = 25;

console.log(name);  // Alice
console.log(age * 2);  // 50

// String concatenation
console.log("User: " + name);  // User: Alice

// Template literals (preferred)
console.log(`User: ${name}, Age: ${age}`);  // User: Alice, Age: 25
```

**Objects and Arrays:**
```javascript
let user = { name: "John", age: 30 };
console.log(user);  // { name: 'John', age: 30 }

let numbers = [1, 2, 3, 4, 5];
console.log(numbers);  // [1, 2, 3, 4, 5]

// Object property expansion in browser DevTools
console.log({ user, numbers });  // Expandable in console
```

**Styled Console (Browser Only):**
```javascript
console.log("%cStyled Text", "color: blue; font-size: 20px; font-weight: bold");

// Multiple styles
console.log(
    "%cError: %cSomething went wrong",
    "color: red; font-weight: bold",
    "color: orange"
);
```

## console.error

Outputs error messages with error styling (typically red).

**Basic Usage:**
```javascript
console.error("This is an error message");
console.error("Error:", "File not found");

// With error object
try {
    throw new Error("Something went wrong");
} catch (e) {
    console.error(e);  // Displays full error with stack trace
}
```

**Practical Examples:**
```javascript
function divide(a, b) {
    if (b === 0) {
        console.error("Error: Division by zero");
        return null;
    }
    return a / b;
}

// API error handling
fetch('/api/data')
    .then(response => response.json())
    .then(data => console.log(data))
    .catch(error => console.error("API Error:", error));
```

**Error vs Log:**
```javascript
// console.log - general information
console.log("User logged in");

// console.error - errors and failures
console.error("Login failed: Invalid credentials");
```

## console.warn

Outputs warning messages with warning styling (typically yellow/orange).

**Basic Usage:**
```javascript
console.warn("This is a warning");
console.warn("Warning: Low disk space");
```

**Use Cases:**
```javascript
// Deprecated feature usage
function oldFunction() {
    console.warn("Warning: oldFunction() is deprecated. Use newFunction() instead");
    // function logic
}

// Validation warnings
function setAge(age) {
    if (age < 0) {
        console.warn("Warning: Age cannot be negative");
        age = 0;
    }
    return age;
}

// Configuration issues
if (!config.apiKey) {
    console.warn("Warning: API key not configured. Using default settings.");
}
```

**Message Severity Hierarchy:**
```javascript
console.log("Info: Operation completed");  // General information
console.warn("Warning: Cache is full");  // Potential issue
console.error("Error: Database connection failed");  // Critical failure
```

## console.table

Displays data in table format, perfect for arrays and objects.

**Array of Objects:**
```javascript
let users = [
    { name: "John", age: 30, role: "Admin" },
    { name: "Jane", age: 25, role: "User" },
    { name: "Bob", age: 35, role: "Moderator" }
];

console.table(users);
// Output: Clean table with columns: (index), name, age, role
```

**Simple Arrays:**
```javascript
let numbers = [10, 20, 30, 40, 50];
console.table(numbers);
// Output: Table with index and value columns

let fruits = ["Apple", "Banana", "Orange"];
console.table(fruits);
```

**Objects:**
```javascript
let user = {
    firstName: "John",
    lastName: "Doe",
    age: 30,
    email: "john@example.com"
};

console.table(user);
// Output: Table with property names and values
```

**Selecting Columns:**
```javascript
let users = [
    { name: "John", age: 30, role: "Admin", email: "john@example.com" },
    { name: "Jane", age: 25, role: "User", email: "jane@example.com" }
];

// Show only specific columns
console.table(users, ["name", "role"]);
```

## console.time / console.timeEnd

Measures execution time of code blocks.

**Basic Timing:**
```javascript
console.time("Operation");
// Code to measure
for (let i = 0; i < 1000000; i++) {
    // Some operation
}
console.timeEnd("Operation");
// Output: Operation: 5.234ms
```

**Multiple Timers:**
```javascript
console.time("Total");
console.time("Step 1");
processStepOne();
console.timeEnd("Step 1");

console.time("Step 2");
processStepTwo();
console.timeEnd("Step 2");

console.timeEnd("Total");
```

**Practical Use:**
```javascript
// Compare algorithm performance
console.time("forEach");
array.forEach(item => process(item));
console.timeEnd("forEach");

console.time("for loop");
for (let i = 0; i < array.length; i++) {
    process(array[i]);
}
console.timeEnd("for loop");

console.time("map");
array.map(item => process(item));
console.timeEnd("map");
```

**console.timeLog (Progress Updates):**
```javascript
console.time("Download");
console.timeLog("Download", "Started");
// Some code
console.timeLog("Download", "50% complete");
// More code
console.timeEnd("Download");
```

## Debugging Output

Techniques for effective debugging with console methods.

**console.dir - Object Inspection:**
```javascript
let element = document.getElementById("myDiv");
console.dir(element);  // Shows all properties of DOM element
```

**console.count - Execution Counter:**
```javascript
function processItem(item) {
    console.count("processItem called");
    // Processing logic
}

processItem(1);  // processItem called: 1
processItem(2);  // processItem called: 2
processItem(3);  // processItem called: 3

console.countReset("processItem called");  // Reset counter
```

**console.trace - Stack Trace:**
```javascript
function third() {
    console.trace("Trace point");
}

function second() {
    third();
}

function first() {
    second();
}

first();
// Shows full call stack: first > second > third
```

**console.assert - Conditional Logging:**
```javascript
let age = 15;
console.assert(age >= 18, "Age must be 18 or older");
// Only logs if condition is false

let users = [];
console.assert(users.length > 0, "Users array is empty", users);
```

**console.group - Grouped Messages:**
```javascript
console.group("User Details");
console.log("Name: John Doe");
console.log("Age: 30");
console.log("Email: john@example.com");
console.groupEnd();

console.groupCollapsed("Collapsed Group");  // Starts collapsed
console.log("Hidden detail 1");
console.log("Hidden detail 2");
console.groupEnd();
```

**console.clear - Clear Console:**
```javascript
console.clear();  // Clears the console
```

## Logging Best Practices

**1. Use Appropriate Log Levels:**
```javascript
// Information
console.log("User logged in");

// Warnings
console.warn("Session expiring soon");

// Errors
console.error("Failed to save data");
```

**2. Descriptive Messages:**
```javascript
// Good: Context and values
console.log("Fetching user data for ID:", userId);

// Avoid: Vague messages
console.log("Fetching");
```

**3. Remove Debug Logs in Production:**
```javascript
// Development
const DEBUG = true;

if (DEBUG) {
    console.log("Debug: Variable value:", someVar);
}

// Or use a logging library with levels
```

**4. Structured Logging:**
```javascript
// Good: Consistent format
console.log("Event:", "UserLogin", "UserID:", userId, "Time:", new Date());

// Better: Object for structure
console.log({
    event: "UserLogin",
    userId: userId,
    timestamp: new Date()
});
```

**5. Use console.table for Data:**
```javascript
// Instead of
console.log(users);

// Use
console.table(users);  // Easier to read
```

**6. Performance Logging:**
```javascript
// Measure expensive operations
console.time("Database Query");
let results = await db.query();
console.timeEnd("Database Query");
```

**7. Don't Log Sensitive Data:**
```javascript
// Never log
console.log("Password:", password);  // Security risk!
console.log("API Key:", apiKey);  // Security risk!

// Instead
console.log("User authenticated");  // Log the action, not the secret
```

**8. Use Environment Variables:**
```javascript
// Only log in development
if (process.env.NODE_ENV === 'development') {
    console.log("Development debug info");
}
```

## Summary / Key Takeaways

- **console.log:** General output; accepts multiple arguments; use template literals for formatting
- **console.error:** Red-styled error messages; use for failures and exceptions
- **console.warn:** Yellow/orange warnings; use for potential issues and deprecations
- **console.table:** Display arrays and objects in clean table format
- **console.time/timeEnd:** Measure code execution time; use labeled timers for multiple measurements
- **console.trace:** Show stack trace for debugging call paths
- **console.count:** Count function calls or iterations
- **console.group:** Organize related messages together
- **Best practices:** Descriptive messages, appropriate log levels, remove debug logs in production, don't log sensitive data
- **Browser DevTools:** Enhanced console features with styling, filtering, and object expansion

The console is your primary tool for debugging and understanding code behavior during development. Use it effectively for faster debugging and better code quality.
