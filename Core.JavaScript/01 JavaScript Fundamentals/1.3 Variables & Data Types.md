# **[[1.3 Variables & Data Types]]**

## Overview

This topic covers the fundamental building blocks of JavaScript programs: variables and data types. You'll learn the three ways to declare variables (var, let, const), understand primitive data types, work with strings, numbers, and booleans, handle undefined and null values, explore the newer symbol and bigint types, and use the typeof operator to check data types.

## Variables: var, let, const

Variables are containers for storing data values. JavaScript provides three keywords for declaring variables.

**`let` - Block-Scoped Variable**
```javascript
let age = 25;
age = 26;  // Can be reassigned
console.log(age);  // 26

// Block scope
if (true) {
    let message = "Hello";
    console.log(message);  // "Hello"
}
// console.log(message);  // Error: message is not defined
```

**`const` - Block-Scoped Constant**
```javascript
const PI = 3.14159;
// PI = 3.14;  // Error: Assignment to constant variable

const user = {name: "John"};
user.name = "Jane";  // OK: Object properties can be modified
// user = {};  // Error: Cannot reassign constant
```

**`var` - Function-Scoped Variable (Legacy)**
```javascript
var count = 10;
count = 11;  // Can be reassigned

// Function scope (not block scope)
if (true) {
    var oldWay = "Accessible outside";
}
console.log(oldWay);  // "Accessible outside" - var leaks out of blocks
```

**When to Use:**
- **`let`** - Use for variables that will change
- **`const`** - Use for variables that won't be reassigned (preferred default)
- **`var`** - Avoid in modern code (use let or const instead)

**Naming Rules:**
- Must start with letter, $, or _
- Can contain letters, digits, $, _
- Cannot use reserved keywords (let, function, class, etc.)
- Case-sensitive (`name` ≠ `Name`)

**Naming Conventions:**
```javascript
// camelCase for variables
let firstName = "John";
let userAge = 30;

// UPPER_CASE for constants
const MAX_SIZE = 100;
const API_KEY = "abc123";

// Descriptive names
let itemCount = 5;  // Good
let ic = 5;  // Avoid single letters except in loops
```

## Primitive Data Types

JavaScript has 7 primitive data types (immutable, not objects):

1. **String** - Text data
2. **Number** - Numeric data
3. **Boolean** - true or false
4. **Undefined** - Uninitialized variable
5. **Null** - Intentional absence of value
6. **Symbol** - Unique identifier (ES6)
7. **BigInt** - Large integers (ES2020)

## Strings

Strings represent text data, enclosed in quotes.

**String Creation:**
```javascript
let single = 'Single quotes';
let double = "Double quotes";
let backticks = `Backticks (template literals)`;

// All three are valid, choose consistently
```

**Template Literals (Backticks):**
```javascript
let name = "Alice";
let age = 28;

// String interpolation
let greeting = `Hello, ${name}! You are ${age} years old.`;
console.log(greeting);  // "Hello, Alice! You are 28 years old."

// Multi-line strings
let multiLine = `This is
a string
spanning multiple lines`;
```

**String Operations:**
```javascript
let text = "JavaScript";

// Length
console.log(text.length);  // 10

// Concatenation
let hello = "Hello" + " " + "World";  // "Hello World"

// Accessing characters
console.log(text[0]);  // "J"
console.log(text.charAt(0));  // "J"

// Common methods
console.log(text.toUpperCase());  // "JAVASCRIPT"
console.log(text.toLowerCase());  // "javascript"
console.log(text.includes("Script"));  // true
console.log(text.slice(0, 4));  // "Java"
```

**Escape Characters:**
```javascript
let escaped = "She said, \"Hello!\"";  // She said, "Hello!"
let newLine = "First line\nSecond line";
let tab = "Name:\tJohn";
let backslash = "C:\\Users\\Desktop";
```

## Numbers

JavaScript has one number type for integers and decimals.

**Number Creation:**
```javascript
let integer = 42;
let decimal = 3.14;
let negative = -10;
let scientific = 2.5e6;  // 2.5 × 10^6 = 2,500,000
```

**Arithmetic:**
```javascript
let a = 10;
let b = 3;

console.log(a + b);  // 13 (addition)
console.log(a - b);  // 7 (subtraction)
console.log(a * b);  // 30 (multiplication)
console.log(a / b);  // 3.333... (division)
console.log(a % b);  // 1 (modulus/remainder)
console.log(a ** b);  // 1000 (exponentiation)
```

**Special Numeric Values:**
```javascript
let infinity = 1 / 0;  // Infinity
let negInfinity = -1 / 0;  // -Infinity
let notANumber = "text" / 2;  // NaN (Not-a-Number)

console.log(isNaN(notANumber));  // true
console.log(isFinite(100));  // true
console.log(isFinite(Infinity));  // false
```

**Number Methods:**
```javascript
let num = 123.456;

console.log(num.toFixed(2));  // "123.46" (string)
console.log(num.toPrecision(4));  // "123.5" (string)
console.log(parseInt("123.45"));  // 123
console.log(parseFloat("123.45"));  // 123.45
```

## Booleans

Booleans represent logical values: `true` or `false`.

**Boolean Creation:**
```javascript
let isActive = true;
let isComplete = false;

// Comparison operations return booleans
let result = 5 > 3;  // true
let equal = 10 === 10;  // true
```

**Boolean in Conditionals:**
```javascript
let loggedIn = true;

if (loggedIn) {
    console.log("Welcome back!");
} else {
    console.log("Please log in");
}
```

**Truthy and Falsy:**

Falsy values (convert to false):
- `false`
- `0`
- `""` (empty string)
- `null`
- `undefined`
- `NaN`

Everything else is truthy:
```javascript
if (0) {
    // Never runs
}

if ("hello") {
    // Always runs (non-empty string is truthy)
}
```

## Undefined vs Null

**`undefined`** - Variable declared but not assigned:
```javascript
let x;
console.log(x);  // undefined
console.log(typeof x);  // "undefined"

function noReturn() {}
console.log(noReturn());  // undefined (functions return undefined by default)
```

**`null`** - Intentional absence of value:
```javascript
let user = null;  // Explicitly set to null
console.log(user);  // null
console.log(typeof null);  // "object" (historical JavaScript quirk)
```

**Comparison:**
```javascript
console.log(undefined == null);  // true (loose equality)
console.log(undefined === null);  // false (strict equality)

// Use cases
let age;  // undefined: not initialized
let selectedItem = null;  // null: intentionally empty
```

## Symbol

Symbols are unique, immutable identifiers (ES6).

**Creating Symbols:**
```javascript
let sym1 = Symbol("description");
let sym2 = Symbol("description");

console.log(sym1 === sym2);  // false (each Symbol is unique)
```

**Use Case - Unique Object Keys:**
```javascript
const ID = Symbol("id");
let user = {
    name: "John",
    [ID]: 12345  // Symbol as property key
};

console.log(user[ID]);  // 12345
// Symbol properties are hidden from normal iteration
```

## BigInt

BigInt handles integers larger than `Number.MAX_SAFE_INTEGER` (2^53 - 1).

**Creating BigInt:**
```javascript
let bigNum = 9007199254740991n;  // Add 'n' suffix
let alsoBig = BigInt("9007199254740991");

console.log(typeof bigNum);  // "bigint"
```

**BigInt Operations:**
```javascript
let a = 10n;
let b = 20n;

console.log(a + b);  // 30n
console.log(a * b);  // 200n

// Cannot mix BigInt and Number
// let mixed = 10n + 5;  // Error
let mixed = 10n + BigInt(5);  // 15n (OK)
```

## typeof Operator

The `typeof` operator returns the data type as a string.

**Syntax:**
```javascript
typeof operand
// or
typeof(operand)
```

**Examples:**
```javascript
console.log(typeof "hello");  // "string"
console.log(typeof 42);  // "number"
console.log(typeof true);  // "boolean"
console.log(typeof undefined);  // "undefined"
console.log(typeof null);  // "object" (bug/quirk)
console.log(typeof Symbol("id"));  // "symbol"
console.log(typeof 10n);  // "bigint"
console.log(typeof {});  // "object"
console.log(typeof []);  // "object" (arrays are objects)
console.log(typeof function(){});  // "function"
```

**Practical Use:**
```javascript
function processValue(value) {
    if (typeof value === "string") {
        console.log("Processing string:", value.toUpperCase());
    } else if (typeof value === "number") {
        console.log("Processing number:", value * 2);
    } else {
        console.log("Unknown type");
    }
}

processValue("hello");  // Processing string: HELLO
processValue(10);  // Processing number: 20
```

## Summary / Key Takeaways

- **Variables:** Use `const` by default, `let` when reassignment needed, avoid `var`
- **Primitive types:** String, Number, Boolean, Undefined, Null, Symbol, BigInt
- **Strings:** Text enclosed in quotes; use backticks for templating and multi-line
- **Numbers:** Single type for integers and decimals; watch for NaN and Infinity
- **Booleans:** true/false; understand truthy/falsy values
- **undefined:** Uninitialized variables; null: intentional absence
- **Symbol:** Unique identifiers for object properties
- **BigInt:** Large integers beyond Number limits
- **typeof:** Check data types; note `typeof null === "object"` quirk

Understanding variables and data types is essential for storing and manipulating data in JavaScript programs.
