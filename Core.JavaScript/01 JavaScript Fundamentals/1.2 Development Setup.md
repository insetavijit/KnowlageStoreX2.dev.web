# **[[1.2 Development Setup]]**

## Overview

This topic covers setting up a professional JavaScript development environment. You'll learn how to use the browser console for quick experimentation, choose and configure code editors, set up VS Code as your primary development tool, install Node.js for server-side JavaScript, master browser DevTools for debugging, and establish an efficient workflow.

## Browser Console

The browser console is your first JavaScript playground—no installation required.

**Accessing the Console:**
- **Chrome/Edge:** Press `F12` or `Ctrl+Shift+J` (Windows/Linux), `Cmd+Option+J` (Mac)
- **Firefox:** Press `F12` or `Ctrl+Shift+K` (Windows/Linux), `Cmd+Option+K` (Mac)
- **Safari:** Enable Developer menu in Preferences, then press `Cmd+Option+C`

**Using the Console:**
```javascript
// Type JavaScript directly in the console
console.log("Hello, JavaScript!");

// Perform calculations
42 + 8

// Declare variables
let name = "Developer";
console.log("Hello, " + name);

// Test functions
function greet(name) {
    return `Hello, ${name}!`;
}
greet("World");
```

**Console Benefits:**
- Instant feedback (REPL: Read-Eval-Print-Loop)
- Quick testing and experimentation
- Debugging live web pages
- No file setup required

## Code Editors

Professional JavaScript development requires a good code editor. Modern editors provide syntax highlighting, auto-completion, error detection, and integrated tools.

**Popular Code Editors:**

**Visual Studio Code (VS Code)** ⭐ Recommended
- Free, open-source
- Extensive extension ecosystem
- Integrated terminal and debugger
- Git integration
- IntelliSense (intelligent code completion)

**Other Options:**
- **WebStorm:** Powerful IDE with advanced features (paid)
- **Sublime Text:** Fast, lightweight editor
- **Atom:** Hackable editor by GitHub (discontinued)
- **Vim/Neovim:** Terminal-based, for advanced users

**Why VS Code?**
- Industry standard for JavaScript development
- Active development and community
- Free with professional features
- Works on Windows, Mac, and Linux

## VS Code Setup

**Installation:**
1. Download from [code.visualstudio.com](https://code.visualstudio.com/)
2. Run installer for your operating system
3. Launch VS Code

**Essential Extensions:**

Install these extensions for JavaScript development:

1. **ESLint** - JavaScript linting and code quality
2. **Prettier - Code formatter** - Automatic code formatting
3. **JavaScript (ES6) code snippets** - Code shortcuts
4. **Path Intellisense** - File path autocomplete
5. **Live Server** - Local development server with live reload
6. **GitLens** - Enhanced Git capabilities

**Installing Extensions:**
- Press `Ctrl+Shift+X` (Windows/Linux) or `Cmd+Shift+X` (Mac)
- Search for extension name
- Click "Install"

**Configuration:**

Create `.vscode/settings.json` in your project:

```json
{
    "editor.formatOnSave": true,
    "editor.tabSize": 2,
    "editor.wordWrap": "on",
    "javascript.suggest.autoImports": true,
    "files.autoSave": "afterDelay",
    "prettier.semi": true,
    "prettier.singleQuote": false
}
```

**Useful Keyboard Shortcuts:**
- `Ctrl+P` / `Cmd+P` - Quick file open
- `Ctrl+Shift+P` / `Cmd+Shift+P` - Command palette
- `Ctrl+B` / `Cmd+B` - Toggle sidebar
- `Ctrl+` ` / `Cmd+` ` - Toggle terminal
- `F5` - Start debugging

## Node.js Installation

Node.js is a JavaScript runtime that lets you run JavaScript outside the browser, essential for modern development workflows.

**Why Install Node.js?**
- Run JavaScript files locally
- Access npm (Node Package Manager) for libraries
- Use build tools (webpack, Babel, etc.)
- Server-side JavaScript development

**Installation Steps:**

1. **Download:** Visit [nodejs.org](https://nodejs.org/)
2. **Choose Version:**
   - LTS (Long Term Support) - Recommended for most users
   - Current - Latest features
3. **Install:** Run the installer with default options
4. **Verify Installation:**

```bash
# Check Node.js version
node --version
# Should output: v20.x.x (or similar)

# Check npm version
npm --version
# Should output: 10.x.x (or similar)
```

**First Node.js Script:**

Create `hello.js`:
```javascript
console.log("Hello from Node.js!");
```

Run it:
```bash
node hello.js
```

**npm Basics:**
```bash
# Initialize a new project
npm init -y

# Install a package
npm install lodash

# Install development dependency
npm install --save-dev eslint

# Run scripts defined in package.json
npm run start
```

## Browser DevTools

Browser Developer Tools are essential for debugging and understanding JavaScript in web pages.

**Key Panels:**

**Console:**
- View logs and errors
- Execute JavaScript
- Inspect variables

**Sources/Debugger:**
- Set breakpoints
- Step through code
- Inspect call stack and scope

**Elements/Inspector:**
- View and modify DOM
- Inspect CSS styles
- Test layout changes

**Network:**
- Monitor HTTP requests
- View request/response data
- Analyze loading performance

**Application/Storage:**
- Inspect localStorage, sessionStorage
- View cookies
- Check service workers

**Performance:**
- Record runtime performance
- Identify bottlenecks
- Analyze memory usage

## Debugging Basics

**Console Logging:**
```javascript
console.log("Basic log");
console.error("Error message");
console.warn("Warning message");
console.table([{name: "John", age: 30}]);
```

**Debugger Statement:**
```javascript
function calculate(x, y) {
    debugger; // Execution pauses here
    return x + y;
}
```

**Breakpoints:**
1. Open DevTools Sources panel
2. Find your JavaScript file
3. Click line number to set breakpoint
4. Refresh page or trigger code
5. Use controls to step through code:
   - **Step over** (F10) - Execute current line
   - **Step into** (F11) - Enter function
   - **Step out** (Shift+F11) - Exit function

**Watch Expressions:**
- Add variables to watch their values
- Evaluate expressions during debugging

## Workflow

**Basic Development Workflow:**

1. **Setup Project:**
   ```bash
   mkdir my-project
   cd my-project
   npm init -y
   code .  # Open VS Code
   ```

2. **Create Files:**
   - `index.html` - HTML structure
   - `script.js` - JavaScript code
   - `style.css` - Styling

3. **Write Code:**
   - Use VS Code with extensions
   - Enable auto-formatting
   - Follow coding standards

4. **Test Locally:**
   - Use Live Server extension
   - Or open `index.html` in browser
   - Check console for errors

5. **Debug:**
   - Use console.log() for quick checks
   - Set breakpoints for complex issues
   - Read error messages carefully

6. **Iterate:**
   - Make changes
   - Refresh browser (or auto-reload)
   - Test again

**Professional Workflow Additions:**
- Version control (Git)
- Code linting (ESLint)
- Code formatting (Prettier)
- Testing frameworks
- Build tools

## Summary / Key Takeaways

- **Browser console** is perfect for quick JavaScript experimentation and learning
- **VS Code** is the industry-standard editor with excellent JavaScript support
- **Essential extensions:** ESLint, Prettier, Live Server enhance productivity
- **Node.js** is required for running JavaScript locally and accessing npm packages
- **Browser DevTools** provide powerful debugging capabilities with console, breakpoints, and performance tools
- **Debugging workflow:** Console logging → Breakpoints → Step through code
- **Professional setup** includes editor, runtime, version control, and quality tools
- **Efficient workflow** involves setup, coding, testing, debugging, and iteration

A proper development environment makes JavaScript development faster, easier, and more enjoyable.
