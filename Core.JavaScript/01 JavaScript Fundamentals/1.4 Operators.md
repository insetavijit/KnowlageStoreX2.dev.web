# **[[1.4 Operators]]**

## Overview

This topic covers JavaScript operators that perform operations on values. You'll learn arithmetic operators for mathematical calculations, comparison operators for evaluating relationships, logical operators for boolean logic, assignment operators for storing values, the ternary operator for inline conditionals, and operator precedence rules that determine evaluation order.

## Arithmetic Operators

Arithmetic operators perform mathematical calculations.

**Basic Arithmetic:**
```javascript
let a = 10;
let b = 3;

console.log(a + b);  // 13 (Addition)
console.log(a - b);  // 7 (Subtraction)
console.log(a * b);  // 30 (Multiplication)
console.log(a / b);  // 3.333... (Division)
console.log(a % b);  // 1 (Modulus - remainder)
console.log(a ** b);  // 1000 (Exponentiation - a^b)
```

**Increment and Decrement:**
```javascript
let count = 5;

// Post-increment: returns value, then increments
let x = count++;  
console.log(x);  // 5
console.log(count);  // 6

// Pre-increment: increments, then returns value
let y = ++count;
console.log(y);  // 7
console.log(count);  // 7

// Decrement works similarly
count--;  // count is now 6
--count;  // count is now 5
```

**String Concatenation:**
```javascript
// + operator also concatenates strings
let greeting = "Hello" + " " + "World";  // "Hello World"
let mixed = "Number: " + 42;  // "Number: 42" (number converts to string)

// Be careful with mixed types
console.log("5" + 3);  // "53" (string concatenation)
console.log("5" - 3);  // 2 (subtraction converts to numbers)
```

## Comparison Operators

Comparison operators evaluate relationships and return boolean values.

**Equality Operators:**
```javascript
// Loose equality (==) - converts types
console.log(5 == "5");  // true (string converted to number)
console.log(true == 1);  // true
console.log(null == undefined);  // true

// Strict equality (===) - no type conversion
console.log(5 === "5");  // false (different types)
console.log(true === 1);  // false
console.log(null === undefined);  // false

// Best practice: always use === for safety
```

**Inequality Operators:**
```javascript
console.log(5 != "5");  // false (loose inequality)
console.log(5 !== "5");  // true (strict inequality)
```

**Relational Operators:**
```javascript
let x = 10;
let y = 20;

console.log(x < y);  // true (less than)
console.log(x > y);  // false (greater than)
console.log(x <= 10);  // true (less than or equal)
console.log(y >= 20);  // true (greater than or equal)

// String comparison (lexicographical)
console.log("apple" < "banana");  // true
console.log("Zebra" < "apple");  // true (uppercase comes first)
```

## Logical Operators

Logical operators combine or invert boolean values.

**AND Operator (&&):**
```javascript
// Returns true only if both operands are true
console.log(true && true);  // true
console.log(true && false);  // false
console.log(false && true);  // false

// Practical use
let age = 25;
let hasLicense = true;
if (age >= 18 && hasLicense) {
    console.log("Can drive");
}

// Short-circuit evaluation
let user = null;
let name = user && user.name;  // null (stops at first falsy value)
```

**OR Operator (||):**
```javascript
// Returns true if at least one operand is true
console.log(true || false);  // true
console.log(false || true);  // true
console.log(false || false);  // false

// Practical use - default values
let userInput = "";
let displayName = userInput || "Guest";  // "Guest"

// Short-circuit evaluation
let value = getValue() || "default";  // Uses first truthy value
```

**NOT Operator (!):**
```javascript
// Inverts boolean value
console.log(!true);  // false
console.log(!false);  // true
console.log(!"");  // true (empty string is falsy)

// Double NOT converts to boolean
console.log(!!"hello");  // true
console.log(!!0);  // false
console.log(!!null);  // false
```

**Nullish Coalescing (??) - ES2020:**
```javascript
// Returns right operand when left is null or undefined
let value1 = null ?? "default";  // "default"
let value2 = 0 ?? "default";  // 0 (0 is not null/undefined)

// Difference from ||
console.log(0 || "fallback");  // "fallback" (0 is falsy)
console.log(0 ?? "fallback");  // 0 (0 is not null/undefined)
```

## Assignment Operators

Assignment operators store values in variables.

**Basic Assignment:**
```javascript
let x = 10;  // Assigns 10 to x
```

**Compound Assignment:**
```javascript
let num = 5;

num += 3;  // num = num + 3; (8)
num -= 2;  // num = num - 2; (6)
num *= 4;  // num = num * 4; (24)
num /= 3;  // num = num / 3; (8)
num %= 3;  // num = num % 3; (2)
num **= 3;  // num = num ** 3; (8)

// String concatenation
let text = "Hello";
text += " World";  // text = text + " World"; ("Hello World")
```

**Logical Assignment (ES2021):**
```javascript
// &&= assigns only if left side is truthy
let a = true;
a &&= false;  // a is now false

// ||= assigns only if left side is falsy
let b = "";
b ||= "default";  // b is now "default"

// ??= assigns only if left side is null/undefined
let c = null;
c ??= "value";  // c is now "value"
```

**Destructuring Assignment:**
```javascript
// Array destructuring
let [first, second] = [1, 2, 3];  // first=1, second=2

// Object destructuring
let {name, age} = {name: "John", age: 30};  // name="John", age=30
```

## Ternary Operator

The ternary operator is a shorthand for if-else conditions.

**Syntax:**
```javascript
condition ? expressionIfTrue : expressionIfFalse
```

**Basic Usage:**
```javascript
let age = 20;
let status = age >= 18 ? "Adult" : "Minor";
console.log(status);  // "Adult"

// Equivalent if-else
let status2;
if (age >= 18) {
    status2 = "Adult";
} else {
    status2 = "Minor";
}
```

**Multiple Conditions:**
```javascript
let score = 85;
let grade = score >= 90 ? "A" :
            score >= 80 ? "B" :
            score >= 70 ? "C" :
            score >= 60 ? "D" : "F";
console.log(grade);  // "B"

// Note: Nested ternaries can reduce readability
// Use if-else for complex logic
```

**Practical Examples:**
```javascript
// Display message based on status
let online = true;
console.log(online ? "User is online" : "User is offline");

// Set default value
let input = "";
let value = input ? input : "No input";

// Conditional function call
let debug = true;
debug ? console.log("Debugging") : null;
```

## Operator Precedence

Operator precedence determines the order of evaluation in expressions.

**Precedence Table (High to Low):**
1. Grouping `()`
2. Unary `++`, `--`, `!`, `typeof`
3. Exponentiation `**`
4. Multiplication, Division, Modulus `*`, `/`, `%`
5. Addition, Subtraction `+`, `-`
6. Comparison `<`, `>`, `<=`, `>=`
7. Equality `==`, `===`, `!=`, `!==`
8. Logical AND `&&`
9. Logical OR `||`
10. Ternary `? :`
11. Assignment `=`, `+=`, `-=`, etc.

**Examples:**
```javascript
// Multiplication before addition
let result1 = 2 + 3 * 4;  // 14 (not 20)

// Use parentheses to override
let result2 = (2 + 3) * 4;  // 20

// Comparison before logical AND
let x = 5;
let check = x > 3 && x < 10;  // true

// Multiple operators
let calc = 10 + 5 * 2 / 5 - 2;  // 10
// Evaluation: 5*2=10, 10/5=2, 10+2=12, 12-2=10
```

**Associativity:**
```javascript
// Left-to-right (most operators)
let a = 10 - 5 - 2;  // 3 (10-5=5, 5-2=3)

// Right-to-left (assignment, exponentiation)
let x, y, z;
x = y = z = 5;  // z=5, y=5, x=5

let power = 2 ** 3 ** 2;  // 512 (3**2=9, 2**9=512)
```

**Best Practices:**
```javascript
// Use parentheses for clarity
let unclear = a + b * c;
let clear = a + (b * c);  // Explicit order

// Break complex expressions
let complex = (a + b) * (c - d) / (e + f);
// Better:
let sum = a + b;
let diff = c - d;
let divisor = e + f;
let result = (sum * diff) / divisor;
```

## Summary / Key Takeaways

- **Arithmetic operators:** `+`, `-`, `*`, `/`, `%`, `**` for mathematical operations
- **Comparison operators:** Use `===` and `!==` for strict equality (avoid `==` and `!=`)
- **Logical operators:** `&&` (AND), `||` (OR), `!` (NOT) for boolean logic; understand short-circuit evaluation
- **Assignment operators:** `=` for assignment; `+=`, `-=`, etc. for compound operations
- **Ternary operator:** `condition ? true : false` for inline conditionals; avoid deep nesting
- **Operator precedence:** Multiplication/division before addition/subtraction; use parentheses for clarity
- **String + number:** `+` concatenates strings, other operators convert to numbers
- **Falsy values:** `false`, `0`, `""`, `null`, `undefined`, `NaN`

Understanding operators is essential for writing expressions and controlling program logic in JavaScript.
