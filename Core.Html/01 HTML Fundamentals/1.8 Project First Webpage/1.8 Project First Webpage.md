| Topic                                          | Focus & Purpose (Researched & Evaluated)                                                                                                                                                                                           |
| ---------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[[1.8.1 Project Planning]]**                 | Establish **project planning process**: defining webpage purpose, identifying content sections, sketching rough layout, gathering text and images, and setting up the project folder structure before coding.                      |
| **[[1.8.2 Creating the HTML File]]**           | Walk through **creating index.html**: generating HTML5 boilerplate (doctype, html, head, body), setting page title, adding meta charset and viewport, saving the file, and opening in browser for first view.                     |
| **[[1.8.3 Building the Header Section]]**      | Construct the **page header**: adding main heading (h1) with name/title, introductory paragraph, possibly a navigation placeholder; establishing the top section that introduces the page content.                                 |
| **[[1.8.4 Adding Main Content]]**              | Develop the **body content areas**: about section with paragraphs, skills or interests lists, multiple headings (h2, h3) for organization, proper content hierarchy; building the core informational sections.                     |
| **[[1.8.5 Incorporating Images]]**             | Add **visual content**: including a profile photo or relevant images, writing descriptive alt text, setting width/height, organizing images in assets folder, and enhancing the page with visual elements.                         |
| **[[1.8.6 Creating Navigation Links]]**        | Build **internal and external links**: linking to sections with fragment identifiers, linking to external sites (GitHub, LinkedIn), email link for contact, proper link text for accessibility.                                    |
| **[[1.8.7 Structuring with Lists]]**           | Implement **list elements**: unordered lists for skills/interests, ordered lists for steps/priorities, nested lists for categories; using lists to organize and present grouped information effectively.                           |
| **[[1.8.8 Adding a Footer]]**                  | Create the **page footer**: copyright notice, contact information, social links, using horizontal rule for separation; completing the page with proper closing content and attribution.                                            |
| **[[1.8.9 Validation and Testing]]**           | Perform **quality assurance**: validating HTML with W3C validator, testing in multiple browsers, checking all links work, verifying images display, fixing any errors; ensuring production-ready code.                             |
| **[[1.8.10 Best Practices Review]]**           | Conduct **final review**: checking semantic element usage, verifying accessibility basics (alt text, link text), code formatting and indentation, commenting key sections, applying lessons from the module.                       |
