|**Subtopic**|**Focus & Purpose**|**Key Concepts / Details**|**One-Line Recall**|
|---|---|---|---|
|**1.6.4 Link Targets**|Explain the `target` attribute|`_self`, `_blank`, `_parent`, `_top`|Controls where the linked document will open.|
|**[[1.6.4.1 _self (Default)]]**|Standard behavior|Same window|Opens in the current tab, replacing current content.|
|**[[1.6.4.2 _blank]]**|New window|New tab/window process|Opens in a fresh tab; keeps original page open.|
|**[[1.6.4.3 Security Risk]]**|The _blank vulnerability|`window.opener` exploits|Target blank can allow the new page to control the previous page.|
|**[[1.6.4.4 The Fix]]**|Security patch|`rel="noopener noreferrer"`|Always add `rel="noopener noreferrer"` when using `target="_blank"`.|
|**[[1.6.4.5 _parent / _top]]**|Frames (Legacy)|Breaking out of iframes|Used to control frame navigation (rarely used now).|
|**[[1.6.4.6 User Expectation]]**|UX rule|Don't force new tabs|Let users decide; only use `_blank` for external links or PDFs.|
