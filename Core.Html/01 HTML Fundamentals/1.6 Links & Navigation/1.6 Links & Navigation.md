| Topic                                       | Focus & Purpose (Researched & Evaluated)                                                                                                                                                                                           |
| ------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[[1.6.1 Anchor Element Basics]]**         | Introduce the **`<a>` element** as the fundamental hyperlink: anatomy of a link (opening tag, content, closing tag), clickable behavior, default styling (blue underline), and the web's interconnected nature through links.     |
| **[[1.6.2 The href Attribute]]**            | Explain **href (hypertext reference)** as the link destination: URL syntax, empty href behavior, omitting href for placeholder links, javascript: URLs (and why to avoid), and how browsers resolve href values.                  |
| **[[1.6.3 Absolute vs Relative URLs]]**     | Distinguish **absolute and relative paths**: full URLs with protocol, domain-relative paths, document-relative paths, parent directory traversal (`../`), when to use each type, and portability considerations.                  |
| **[[1.6.4 Link Targets]]**                  | Explain the **target attribute**: `_self` (default), `_blank` for new tabs, `_parent` and `_top` for frames; security implications of `_blank`; using `rel="noopener noreferrer"` for safe external links.                        |
| **[[1.6.5 Email Links (mailto)]]**          | Cover **mailto: protocol** for email links: basic syntax, adding subject and body parameters, multiple recipients (cc, bcc), URL encoding special characters, and user experience considerations.                                 |
| **[[1.6.6 Telephone Links (tel)]]**         | Explain **tel: protocol** for phone links: mobile click-to-call functionality, proper phone number formatting, international format considerations, and enhancing mobile user experience with callable numbers.                   |
| **[[1.6.7 Fragment Identifiers]]**          | Detail **in-page navigation** with fragments: creating anchor targets with id attributes, linking with `#id`, smooth scrolling behavior, linking to fragments on other pages, and table of contents patterns.                     |
| **[[1.6.8 Download Links]]**                | Explain the **download attribute**: triggering file downloads instead of navigation, specifying suggested filenames, file type considerations, and combining with href for downloadable resources.                                |
| **[[1.6.9 Link Accessibility]]**            | Cover **accessible link practices**: descriptive link text (avoid "click here"), title attribute usage, focus states for keyboard users, aria-label for icon links, and screen reader considerations.                             |
| **[[1.6.10 Navigation Patterns]]**          | Introduce **common navigation structures**: horizontal nav bars, vertical menus, breadcrumbs, skip links for accessibility; preparing for the semantic `<nav>` element covered in later sections.                                 |
