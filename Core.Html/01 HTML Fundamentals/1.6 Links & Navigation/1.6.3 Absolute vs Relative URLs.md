|---|---|---|---|
|**1.6.3 Absolute vs Relative URLs**|Distinguish path types|Full domain vs Local path, Portability|Absolute is a full address; Relative is directions from here.|
|**[[1.6.3.1 Absolute URLs]]**|External links|`https://google.com`|Contains protocol, domain, and path; works from anywhere.|
|**[[1.6.3.2 Relative URLs]]**|Internal links|`./about.html`|Path is calculated relative to the current file's location.|
|**[[1.6.3.3 Directory Traversal]]**|Navigation|`../` (Parent), `./` (Current), `/` (Root)|`../` moves you up one folder level.|
|**[[1.6.3.4 Root Relative]]**|Server paths|`/images/logo.png`|Starts from the website's root folder, ignoring current depth.|
|**[[1.6.3.5 When to use which]]**|Use case|External vs Internal|Use absolute for other sites; use relative for your own site.|
|**[[1.6.3.6 Portability]]**|Dev vs Prod|Benefits of relative|Relative paths work on localhost and production without changing code.|
