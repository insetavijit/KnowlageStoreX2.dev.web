|**Subtopic**|**Focus & Purpose**|**Key Concepts / Details**|**One-Line Recall**|
|---|---|---|---|
|**1.2.8 Viewport Meta Tag**|Detail the directive controlling layout on mobile devices|`width=device-width`, `initial-scale`, scaling constraints|The viewport tag tells the browser how to fit the page on a mobile screen.|
|**[[1.2.8.1 The "Mobile First" Key]]**|Why it's meaningful|Historical mobile rendering (zoomed out) vs modern RWD|Without it, mobile browsers render pages like desktop sites at 980px wide.|
|**[[1.2.8.2 Width Property]]**|Setting the stage|`width=device-width`|Sets the width of the viewport to match the actual device screen width.|
|**[[1.2.8.3 Scale Properties]]**|Zoom control|`initial-scale=1.0`, `maximum-scale`, `user-scalable`|Controls the initial zoom level and whether users can pinch-to-zoom.|
|**[[1.2.8.4 Accessibility Implications]]**|Zoom locking dangers|Avoid `user-scalable=no`|Never prevent users from zooming; it violates accessibility standards.|
|**[[1.2.8.5 Common Configurations]]**|Standard snippet|The ubiquitous boilerplate used in every RWD site|`<meta name="viewport" content="width=device-width, initial-scale=1.0">`|
|**[[1.2.8.6 Viewport Units]]**|CSS relationship|`vw`, `vh`, `vmin`, `vmax`|The meta tag sets the reference frame for CSS viewport units.|
