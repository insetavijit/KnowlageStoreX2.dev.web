|**1.2.6 The Body Section**|Define the container for visible content|Visible viewport, block formatting context, scripting host|The `<body>` contains everything the user actually sees on the page.|
|**[[1.2.6.1 Visible Content]]**|Relationship to viewport|Rendering area, overflow handling|If it's not in the body, it's not on the screen.|
|**[[1.2.6.2 Content Categories]]**|What goes inside|Flow content, phrasing content, heading content|Almost all HTML elements (div, p, img, button) live here.|
|**[[1.2.6.3 Script Placement]]**|Performance optimization|End of body vs Head + Defer|Scripts strictly related to UI manipulation often sit at the end of body.|
|**[[1.2.6.4 Body Styling]]**|CSS defaults|Backgrounds, default margins (8px), font inheritance|The body is the primary hook for global page styles like fonts and backgrounds.|
|**[[1.2.6.5 One Body Rule]]**|Structure validity|Single body per document|A valid HTML document can have only one body element.|
|**[[1.2.6.6 Events]]**|Global event handlers|`onload`, `onresize`, window interaction|The body often captures global page events.|
