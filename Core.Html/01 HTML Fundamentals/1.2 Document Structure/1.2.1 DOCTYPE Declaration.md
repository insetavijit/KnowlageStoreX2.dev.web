|**1.2.1 DOCTYPE Declaration**|Explain the purpose of the Document Type Declaration|Standards Mode vs Quirks Mode, `<!DOCTYPE html>`, Historical DTDs|The DOCTYPE is a required preamble that tells browsers to render in standards mode.|
|**[[1.2.1.1 Purpose of DOCTYPE]]**|Why it exists|Browser rendering modes, validation, not an HTML tag|It switches the browser engine from legacy compatibility to modern standards.|
|**[[1.2.1.2 The Content Layout]]**|Legacy vs Modern Syntax|SGML-based DTDs (e.g., HTML 4.01 Strict) vs HTML5's simplified version|HTML5 replaced 3 lines of complex URI references with `<!DOCTYPE html>`.|
|**[[1.2.1.3 Standards vs Quirks Mode]]**|Rendering implications|Box model differences, layout bugs, "Almost Standards" mode|Without a DOCTYPE, browsers emulate old buggy behavior (Quirks Mode).|
|**[[1.2.1.4 Case Insensitivity]]**|Syntax rules|Case-insensitive nature of the declaration|`<!doctype html>` is just as valid as `<!DOCTYPE html>`.|
|**[[1.2.1.5 Where it Goes]]**|Placement rules|First line of code, before `<html>`, no whitespace before it|It must be the absolute first thing in the file.|
|**[[1.2.1.6 Common Mistakes]]**|Troubleshooting|Missing DOCTYPE, wrong DTD, whitespace errors|Forgetting the DOCTYPE is the #1 cause of CSS layout issues.|
