|**1.3.5 File Organization Basics**|Establish project structure fundamentals|Root folder, assets directory, index.html|A disorganized project is a nightmare to maintain.|
|**[[1.3.5.1 The Root Directory]]**|Project container|The top-level folder of your website|All website files must live inside one main folder.|
|**[[1.3.5.2 index.html]]**|The entry point|Default page served by web servers|Main pages must be named `index.html` to load automatically.|
|**[[1.3.5.3 Asset Management]]**|Subfolders|`css/`, `js/`, `images/`, `fonts/`|Keep different file types in their own dedicated folders.|
|**[[1.3.5.4 Relative Paths]]**|Linking internal files|`./`, `../`, path resolution|Understanding how to navigate folders is key to linking files.|
|**[[1.3.5.5 Separation of Concerns]]**|File types|HTML, CSS, and JS files|Don't put CSS or JS inside your HTML file; separate them.|
|**[[1.3.5.6 Case Sensitivity]]**|Server behavior|Linux vs Windows file systems|Always assume file paths are case-sensitive (use lowercase).|
