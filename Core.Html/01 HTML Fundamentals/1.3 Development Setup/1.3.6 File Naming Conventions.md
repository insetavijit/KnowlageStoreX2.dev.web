|**1.3.6 File Naming Conventions**|Define naming best practices|Lowercase, hyphens, semantic names|Good filenames are lowercase, descriptive, and use hyphens.|
|**[[1.3.6.1 Lowercase Only]]**|Compatibility rule|Avoid mixed case `MyFile.html`|Servers treat `File.html` and `file.html` differently; stick to lowercase.|
|**[[1.3.6.2 Hyphens vs Underscores]]**|Readability and SEO|`my-file.html` vs `my_file.html`|Google prefers hyphens; they are treated as space characters.|
|**[[1.3.6.3 No Spaces]]**|URL safety|Escaping issues (`My%20File.html`)|Never use spaces in filenames; they break URLs.|
|**[[1.3.6.4 Semantic Naming]]**|Descriptiveness|`about.html` vs `page2.html`|The filename should tell you what is inside the file.|
|**[[1.3.6.5 Special Characters]]**|Forbidden characters|Avoid `?`, `&`, `%`, `#`, `:`, `*`|Stick to letters, numbers, hyphens, and underscores only.|
|**[[1.3.6.6 Extension consistency]]**|File types|`.html` vs `.htm`|Always use `.html` for consistency.|
