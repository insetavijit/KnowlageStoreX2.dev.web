|**Subtopic**|**Focus & Purpose**|**Key Concepts / Details**|**One-Line Recall**|
|---|---|---|---|
|**1.1.5 Core HTML Document Structure**|Define the basic anatomy of an HTML document|Doctype, root, head, body, metadata|Every HTML page requires a standard structural foundation.|
|**[[1.1.5.1 The Doctype Declaration]]**|Inform the browser of document mode|`<!DOCTYPE html>`, Standards Mode vs Quirks Mode|The Doctype triggers standard rendering mode.|
|**[[1.1.5.2 The Root Element]]**|The container for all page content|`<html>`, `lang` attribute (accessibility/SEO)|The html tag is the root of the document tree.|
|**[[1.1.5.3 The Head Section]]**|Metadata container (not displayed)|`<head>`, `<title>`, `<meta>`, `<link>`, `<script>`|The head contains machine-readable data about the page.|
|**[[1.1.5.4 The Body Section]]**|Content container (displayed)|`<body>`, visible elements, flow content|The body contains everything the user sees.|
|**[[1.1.5.5 Character Encoding]]**|Ensuring text renders correctly|`<meta charset="UTF-8">`, UTF-8 universal standard|Always use UTF-8 to prevent Mojibake (garbled text).|
|**[[1.1.5.6 The Viewport Meta Tag]]**|Enabling mobile responsiveness|`width=device-width`, `initial-scale=1.0`|The viewport tag tells mobile browsers how to scale the page.|
