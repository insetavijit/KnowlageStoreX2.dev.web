| **[[1.1.1 What Is HTML]]**                   | Explain HTML as the foundational **markup language** of the Web that structures content for browsers; clarify the distinction between markup and programming and how HTML annotates text/elements for rendering. |
| **[[1.1.2 History of HTML]]**                | Trace the **evolution of HTML** from its early SGML roots to HTML5 and the modern HTML living standard; highlight key milestones that shaped today’s Web.                                                        |
| **[[1.1.3 HTML vs XHTML]]**                  | Compare the **syntax rules, parsing models, and use cases** of HTML vs. XHTML; emphasize why XHTML’s strict XML-based rules differ from HTML’s lenient, browser-friendly model.                                  |
| **[[1.1.4 HTML5 Overview]]**                 | Summarize HTML5 enhancements over earlier HTML: **new semantic elements, multimedia support without plugins, richer form controls, offline capabilities, and APIs** for modern apps.                             |
| **[[1.1.5 Core HTML Document Structure]]**   | Define the **basic anatomy of an HTML document** (doctype, `<html>`, `<head>`, `<body>`, metadata) and how these parts inform browser interpretation and accessibility.                                          |
| **[[1.1.6 How Browsers Work (Basics)]]**     | Explain the **browser rendering pipeline**: parsing HTML into a DOM tree, then rendering the visual layout; illustrate how HTML elements drive browser display. (Industry learning outcome)                      |
| **[[1.1.7 Why Learn HTML]]**                 | Establish the **practical importance** of HTML for all web development directions, including accessibility, SEO, and ability to build structured, browser-compatible content.                                    |
| **[[1.1.8 HTML in the Web Stack]]**          | Position HTML relative to **CSS and JavaScript** in the front-end stack; clarify static vs dynamic roles and how HTML integrates with styling/behavior layers.                                                   |
| **[[1.1.9 Common Beginner Misconceptions]]** | Address frequent misunderstandings (HTML _is not_ a programming language; HTML _does not_ control style or behavior alone) to reinforce correct conceptual models. (Derived from common curriculum practice)     |
