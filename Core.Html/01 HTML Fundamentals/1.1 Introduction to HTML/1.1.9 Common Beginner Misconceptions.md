|**1.1.9 Common Beginner Misconceptions**|Address frequent misunderstandings to reinforce correct models|Not programming, not for style, not just visual|Clearing up myths prevents bad habits early on.|

|**[[1.1.9.3 "Div Soup"]]**|The overuse of generic containers|`<div>` vs `<section>`, bloat, accessibility harm|Overusing divs creates messy, meaningless code structures.|
|**[[1.1.9.4 "Visual Order = Source Order"]]**|Understanding flow vs position|Flexbox/Grid reordering, screen reader confusion, tab order|What you see isn't always what the screen reader says.|
|**[[1.1.9.5 "Validation Doesn't Matter"]]**|The danger of browser error correction|Quirks mode, unpredictable rendering, future proofing|Just because it "looks right" doesn't mean the code is valid.|
|**[[1.1.9.6 Block vs Inline"]]**|The old layout model vs modern CSS|Display property, flow layout, default behaviors|HTML elements have default display types that CSS changes.|
