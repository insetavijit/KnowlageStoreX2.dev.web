|**1.4.9 Generic Containers (div)**|Introduce the generic block container|`<div>`, No semantic value, Layout hooks|The `<div>` is a box with no meaning, just distinct boundaries.|
|**[[1.4.9.1 Purpose]]**|Grouping content|Wrappers, Containers, Grids|Use it when no other semantic element applies.|
|**[[1.4.9.2 The "Divitis" Problem]]**|Overuse|Accessibility noise, messy code|Don't use a div when a `<section>` or `<article>` is more appropriate.|
|**[[1.4.9.3 Styling Hooks]]**|CSS target|`class="card"`, `id="hero"`|Divs are primarily used as targets for CSS styling.|
|**[[1.4.9.4 Layout Grid]]**|Structure|Flexbox/Grid parents|Divs often serve as the invisible rows and columns of a layout.|
|**[[1.4.9.5 Semantic alternative]]**|Better choices|header, main, footer|Always ask: "Is there a better tag than div?" before typing it.|
