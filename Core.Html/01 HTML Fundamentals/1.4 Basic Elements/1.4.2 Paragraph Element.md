|**1.4.2 Paragraph Element**|Define the fundamental text container|`<p>`, Block-level, Automatic margins|The paragraph element is the default container for body text.|
|**[[1.4.2.1 Block Behavior]]**|Layout default|Starts on new line, full width|Paragraphs automatically stack vertically.|
|**[[1.4.2.2 Browser Margins]]**|Default styling|User Agent stylesheet values (usually 1em)|Browsers add space above and below paragraphs by default.|
|**[[1.4.2.3 Permitted Content]]**|Nesting rules|Phrasing content only (no blocks inside)| You cannot put a `<div>` or heading inside a `<p>`.|
|**[[1.4.2.4 Empty Paragraphs]]**|Antipattern|Using `<p></p>` for spacing|Don't use empty paragraphs for vertical space; use CSS margins.|
|**[[1.4.2.5 Text Normalization]]**|Rendering behavior|Whitespace collapsing|Multiple spaces and newlines in code become a single space on screen.|
|**[[1.4.2.6 Semantic Value]]**|Meaning|Grouping related sentences|It tells the browser "this block of text belongs together".|
