| Topic                                        | Focus & Purpose (Researched & Evaluated)                                                                                                                                                                                           |
| -------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[[3.2.1 Header Element]]**                 | Define the **`<header>` element** for introductory content: page headers with logo/nav, article headers with title/byline, multiple headers per page validity, and distinguishing from the head element.                          |
| **[[3.2.2 Nav Element]]**                    | Explain the **`<nav>` element** for navigation sections: primary navigation, table of contents, pagination, breadcrumbs; when nav is appropriate vs simple link lists, and multiple nav elements per page.                        |
| **[[3.2.3 Main Element]]**                   | Detail the **`<main>` element** for primary content: one per page rule, excludes repeated content (nav, footer), skip-link target, and how main identifies the unique content of each page.                                       |
| **[[3.2.4 Footer Element]]**                 | Define the **`<footer>` element** for closing content: page footers with copyright/links, article footers with metadata/author, multiple footers validity, and typical footer content patterns.                                   |
| **[[3.2.5 Section Element]]**                | Explain the **`<section>` element** for thematic groupings: content with its own heading, chapters in documentation, tabbed content panels, distinguishing section from div and article.                                          |
| **[[3.2.6 Article Element]]**                | Detail the **`<article>` element** for self-contained content: blog posts, news articles, forum posts, comments, widgets; the "independently distributable" test for determining article appropriateness.                         |
| **[[3.2.7 Aside Element]]**                  | Define the **`<aside>` element** for tangentially related content: sidebars, pull quotes, advertising, related links; content that could be removed without affecting main content comprehension.                                 |
| **[[3.2.8 Landmark Roles Mapping]]**         | Explain how **semantic elements map to ARIA landmarks**: header→banner, nav→navigation, main→main, footer→contentinfo; why native semantics preferred over explicit role attributes.                                              |
| **[[3.2.9 Nesting Rules]]**                  | Cover **valid nesting patterns**: article within section, section within article, header/footer within article; invalid patterns to avoid, and maintaining logical document hierarchy.                                            |
| **[[3.2.10 Page Layout Example]]**           | Demonstrate a **complete semantic page structure**: combining all structural elements, proper nesting, appropriate heading levels, and how the elements work together to create accessible, meaningful layout.                    |
