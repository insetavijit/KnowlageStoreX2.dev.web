| Topic                                        | Focus & Purpose (Researched & Evaluated)                                                                                                                                                                                           |
| -------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[[9.1.1 What Is Accessibility]]**          | Define **web accessibility (a11y)**: making web content usable by people with disabilities, the broad range of disabilities addressed, and access as a right not a feature.                                                       |
| **[[9.1.2 Types of Disabilities]]**          | Cover **disability categories**: visual (blindness, low vision, color blindness), auditory (deafness, hard of hearing), motor (limited mobility), cognitive (learning disabilities, attention), and temporary/situational.        |
| **[[9.1.3 WCAG Guidelines Overview]]**       | Introduce **Web Content Accessibility Guidelines**: WCAG 2.1/2.2, A/AA/AAA conformance levels, and WCAG as the global accessibility standard.                                                                                     |
| **[[9.1.4 POUR Principles]]**                | Explain the **four WCAG principles**: Perceivable, Operable, Understandable, Robust; what each means, and POUR as accessibility foundation.                                                                                       |
| **[[9.1.5 Legal Requirements]]**             | Cover **accessibility laws**: ADA, Section 508, European Accessibility Act, lawsuits and enforcement, and legal obligation for accessibility.                                                                                     |
| **[[9.1.6 Assistive Technologies]]**         | Introduce **assistive tech**: screen readers (NVDA, VoiceOver, JAWS), screen magnifiers, alternative input devices, voice control, and how users access the web differently.                                                      |
| **[[9.1.7 Business Case for Accessibility]]**| Explain **benefits beyond compliance**: larger audience reach, SEO improvements, better usability for all, innovation driver, and accessibility as good business.                                                                 |
| **[[9.1.8 Common Accessibility Barriers]]**  | Identify **frequent accessibility problems**: missing alt text, keyboard traps, poor color contrast, lack of captions, complex navigation, and awareness of what to avoid.                                                        |
| **[[9.1.9 Accessibility vs Usability]]**     | Clarify **accessibility's relationship to UX**: overlap with usability, accessibility as baseline, designing for extremes benefits everyone, and accessible design as good design.                                                |
| **[[9.1.10 Starting the Accessibility Journey]]** | Provide **first steps for accessibility**: learning mindset, building awareness, integrating accessibility into workflow, and approaching accessibility as ongoing practice.                                                  |
