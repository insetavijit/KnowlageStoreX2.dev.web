| Topic                                        | Focus & Purpose (Researched & Evaluated)                                                                                                                                                                                           |
| -------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[[4.1.1 Form Element Basics]]**            | Introduce the **`<form>` element** as the data collection container: form boundaries, automatic data grouping, default behaviors, and how forms enable user input submission to servers.                                          |
| **[[4.1.2 Action Attribute]]**               | Explain the **action attribute** for submission destination: URL endpoints, relative vs absolute paths, empty action (submit to current page), and how action defines where form data is sent.                                    |
| **[[4.1.3 Method Attribute]]**               | Detail **GET vs POST methods**: GET for data retrieval (visible in URL), POST for data submission (in request body), security implications, when to use each method, and RESTful considerations.                                  |
| **[[4.1.4 Input Element Basics]]**           | Introduce the **`<input>` element**: self-closing syntax, type attribute fundamentals, name attribute for data identification, value attribute, and input as the primary form control.                                            |
| **[[4.1.5 Label Element]]**                  | Detail the **`<label>` element** for input identification: explicit (for attribute) vs implicit (wrapping) association, click-to-focus behavior, accessibility requirement, and why every input needs a label.                    |
| **[[4.1.6 Submit Buttons]]**                 | Cover **form submission controls**: `<input type="submit">`, `<button type="submit">`, button element advantages, default button behavior in forms, and styling submit controls.                                                  |
| **[[4.1.7 Form Structure Best Practices]]**  | Establish **form organization patterns**: logical field grouping, consistent layout, progressive disclosure, form length considerations, and creating user-friendly form experiences.                                             |
| **[[4.1.8 Name Attribute Importance]]**      | Explain the **name attribute** for data transmission: how names become keys in form data, naming conventions, array-style names for multiple values, and debugging form submissions.                                              |
| **[[4.1.9 Enctype Attribute]]**              | Detail **form encoding types**: application/x-www-form-urlencoded (default), multipart/form-data (for files), text/plain; when each encoding is required and what they mean for data transmission.                                |
| **[[4.1.10 Form vs Non-Form Inputs]]**       | Distinguish **form-connected vs standalone inputs**: inputs outside forms, JavaScript-driven forms, client-side-only inputs, and understanding when form element is necessary vs optional.                                        |
