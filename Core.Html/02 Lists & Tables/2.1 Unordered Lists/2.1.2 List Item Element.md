|**Subtopic**|**Focus & Purpose**|**Key Concepts / Details**|**One-Line Recall**|
|---|---|---|---|
|**2.1.2 List Item Element**|Explain the `<li>` element|Individual entries, nesting content|The `<li>` tag defines a single item within a list.|
|**[[2.1.2.1 Syntax]]**|Required child|Must be inside `<ul>`, `<ol>`, or `<menu>`|You cannot use `<li>` without a parent list container.|
|**[[2.1.2.2 Content Model]]**|Flow content|Block items allowed|You can put almost anything inside an `<li>`: headings, images, or even other lists.|
|**[[2.1.2.3 The Marker]]**|Visual|Bullet/Number|The `<li>` element is what actually gets the bullet point or number attached to it.|
|**[[2.1.2.4 Display Type]]**|`display: list-item`|Hybrid|It acts like a block element but has an extra "marker box".|
|**[[2.1.2.5 Closing Tag]]**|Validity|`</li>`|While strictly optional in HTML5 (auto-closes), always write `</li>` to avoid bugs.|
|**[[2.1.2.6 Styling]]**|Targeting|Individual control|You can style specific items differently (e.g., active menu item).|
