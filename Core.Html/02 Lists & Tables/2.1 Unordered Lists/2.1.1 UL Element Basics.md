|**Subtopic**|**Focus & Purpose**|**Key Concepts / Details**|**One-Line Recall**|
|---|---|---|---|
|**2.1.1 UL Element Basics**|Introduce the `<ul>` element|Grouping non-sequential items|Use `<ul>` when the order of the items doesn't change the meaning.|
|**[[2.1.1.1 Semantic Meaning]]**|No inherent order|Groceries, Features, Links|The collection is a "set", not a "sequence".|
|**[[2.1.1.2 Syntax]]**|Container|`<ul>` wraps `<li>`|The `<ul>` element can *only* contain `<li>` elements as direct children.|
|**[[2.1.1.3 Visual Default]]**|Bullets|Disc markers|Browsers add bullet points to unordered lists by default.|
|**[[2.1.1.4 Block Level]]**|Layout|Full width, new line|The `<ul>` is a block-level element, taking up the full width of its container.|
|**[[2.1.1.5 Indentation]]**|Spacing|Padding/Margin|Browsers add left padding (usually 40px) to make room for bullets.|
|**[[2.1.1.6 Interaction]]**|Grouping for JS/CSS|Handling sets|It allows you to style or manipulate the entire group of items as one unit.|
