|**2.7.8 Sticky Headers and Columns**|Navigation aid|`position: sticky`, `z-index`|Keep definitions visible while exploring deep data.|
|**[[2.7.8.1 Vertical Sticky]]**|Header|`top: 0`|Pinning the `<thead>` to the top of the viewport/container.|
|**[[2.7.8.2 Horizontal Sticky]]**|Column|`left: 0`|Pinning the first column (identifier) to the left side.|
|**[[2.7.8.3 Z-Index War]]**|Stacking|Layers|Headers need higher z-index than data; intersection (top-left) needs highest.|
|**[[2.7.8.4 Container]]**|Constraint|Overflow|Sticky elements react to their scrolling ancestor (often the table wrapper).|
|**[[2.7.8.5 Background]]**|Visual|Opacity|Sticky elements need an opaque background, or the data will scroll *under* them visibly.|
