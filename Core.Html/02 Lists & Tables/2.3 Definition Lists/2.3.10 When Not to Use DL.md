|**Subtopic**|**Focus & Purpose**|**Key Concepts / Details**|**One-Line Recall**|
|---|---|---|---|
|**2.3.10 When Not to Use DL**|Clarify anti-patterns|Dialogs, Layouts, Standard text|Don't use it for things that aren't strictly definitions or pairs.|
|**[[2.3.10.1 Dialogs]]**|Scripts|Chat logs|Don't use DL for scripts ("Hamlet: To be..."); use `<p>` with `<b>` or other text tags.|
|**[[2.3.10.2 Layout Only]]**|Indentation|Visuals|Never use DL just because you want indentation; use CSS margins.|
|**[[2.3.10.3 Form Labels]]**|Forms|Inputs|Don't use DL for forms; use `<label>` and `<input>`.|
|**[[2.3.10.4 Headers]]**|Headings|Sections|Don't use DT as a fake heading; use `<h1>-<h6>`.|
|**[[2.3.10.5 Single Items]]**|Overkill|One pair|If you have just one Key-Value pair, a DL is probably too heavy.|
|**[[2.3.10.6 Semantics First]]**|Rule|Meaning|Only use it if the relationship is truly "Term -> Description".|
