|**Subtopic**|**Focus & Purpose**|**Key Concepts / Details**|**One-Line Recall**|
|---|---|---|---|
|**2.5.4 Thead Element**|Introduce header grouping|Semantic head, Sticky headers, Printing|Groups the header rows separately from the data rows.|
|**[[2.5.4.1 Semantic Meaning]]**|Structure|Head of the body|Tells the browser "These rows are the titles".|
|**[[2.5.4.2 Printing]]**|Behavior|Repetition|Browsers can repeat the `<thead>` at the top of every page if the table prints across pages.|
|**[[2.5.4.3 Styling]]**|Hook|Targeting|Easy to style the whole header block (e.g., dark background) with `thead { ... }`.|
|**[[2.5.4.4 Sticky Header]]**|CSS|`position: sticky`|The primary reason to use `<thead>` today is to enable sticky headers while scrolling.|
|**[[2.5.4.5 Placement]]**|Order|First|Must appear before `<tbody>` and `<tfoot>`.|
|**[[2.5.4.6 Content]]**|Rows|`<tr><th>`|Usually contains one or two rows of headers.|
