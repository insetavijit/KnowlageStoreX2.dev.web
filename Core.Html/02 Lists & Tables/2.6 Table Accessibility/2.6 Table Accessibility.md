| **[[2.6.1 Why Table Accessibility Matters]]**| Establish the **importance of accessible tables**: screen reader users navigate differently, complex tables are confusing without proper markup, legal requirements (WCAG), and table accessibility as inclusion.                 |
| **[[2.6.2 Scope Attribute]]**               | Detail the **scope attribute** on th elements: `scope="col"` for column headers, `scope="row"` for row headers, `scope="colgroup"` and `scope="rowgroup"` for spanning headers; explicit header-cell relationships.               |
| **[[2.6.3 Headers Attribute]]**             | Explain the **headers attribute** on td elements: explicitly associating cells with header ids, required for complex tables with multiple headers, space-separated id lists, and when headers is necessary vs scope.              |
| **[[2.6.4 ID-Headers Relationship]]**       | Detail **id assignment to headers**: giving th elements unique ids, referencing multiple ids in headers attribute, building explicit cell-to-header mappings for complex tables with irregular structures.                        |
| **[[2.6.5 Caption for Context]]**           | Reinforce **caption's accessibility role**: providing table purpose to screen readers, announced before table navigation, descriptive captions vs decorative titles, and caption as the table's accessible name.                  |
| **[[2.6.6 Screen Reader Navigation]]**      | Explain **how screen readers traverse tables**: table mode navigation, moving by row/column, hearing header announcements with each cell, and how proper markup creates coherent reading experience.                              |
| **[[2.6.7 ARIA for Tables]]**               | Cover **ARIA attributes for tables**: role="table" (when needed), aria-label, aria-describedby, aria-labelledby; when ARIA supplements vs when native HTML semantics suffice.                                                     |
| **[[2.6.8 Summary Attribute (Legacy)]]**    | Address the **deprecated summary attribute**: historical purpose, why removed from HTML5, modern alternatives (caption, aria-describedby), and handling legacy content with summary.                                              |
| **[[2.6.9 Testing Table Accessibility]]**   | Provide **accessibility testing methods**: screen reader testing (NVDA, VoiceOver), automated tools (axe, WAVE), keyboard navigation testing, and verifying header associations work correctly.                                   |
| **[[2.6.10 Accessible Table Checklist]]**   | Summarize **table accessibility requirements**: th for headers, scope/headers attributes, caption present, no layout tables, logical reading order; quick reference for validating table accessibility.                           |
