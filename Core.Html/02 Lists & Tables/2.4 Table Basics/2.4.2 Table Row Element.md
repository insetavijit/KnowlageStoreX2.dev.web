|---|---|---|---|
|**2.4.2 Table Row Element**|Explain the `<tr>` element|Rows, Horizontal grouping|The `<tr>` defines a single horizontal row of cells.|
|**[[2.4.2.1 Syntax]]**|Container|Wraps `<td>` or `<th>`|A table is a stack of `<tr>` elements.|
|**[[2.4.2.2 Grid Logic]]**|Structure|Row-first|HTML tables are row-oriented; you define rows, not columns.|
|**[[2.4.2.3 Content Model]]**|Strict|Cells only|A `<tr>` can *only* contain `<td>` or `<th>` elements directly.|
|**[[2.4.2.4 Display]]**|`display: table-row`|Behavior|Rows don't have margins or padding in the same way block elements do.|
|**[[2.4.2.5 Styling]]**|Backgrounds|Striping|You can style a whole row (e.g., background color) to affect all cells inside it.|
|**[[2.4.2.6 Missing Rows]]**|Validation|Broken grids|If one row has fewer cells than others, the table can look broken.|
