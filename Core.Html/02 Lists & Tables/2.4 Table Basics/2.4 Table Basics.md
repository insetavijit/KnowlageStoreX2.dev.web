| Topic                                    | Focus & Purpose (Researched & Evaluated)                                                                                                                                                                                           |
| ---------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[[2.4.1 Table Element Basics]]**       | Introduce the **`<table>` element** as the tabular data container: semantic meaning for two-dimensional data, block-level behavior, historical misuse for layout, and modern appropriate usage.                                   |
| **[[2.4.2 Table Row Element]]**          | Explain the **`<tr>` element** as the table row container: required parent for td/th cells, defines horizontal groupings, how rows create the data grid structure, and proper nesting within table.                               |
| **[[2.4.3 Table Data Cell]]**            | Detail the **`<td>` element** as the data cell: contains actual table values, block-level content allowed, default left alignment, relationship to row and column positioning in the grid.                                        |
| **[[2.4.4 Table Header Cell]]**          | Explain the **`<th>` element** as the header cell: semantic distinction from td, default bold/centered styling, scope attribute introduction, critical role for table accessibility and comprehension.                            |
| **[[2.4.5 Simple Table Structure]]**     | Demonstrate **basic table anatomy**: table → tr → th (headers) → tr → td (data); creating a minimal valid table, understanding the row-column grid, and visual representation of structure.                                       |
| **[[2.4.6 Column Headers]]**             | Cover **horizontal headers**: th elements in first row, labeling columns, scope="col" for accessibility, and how column headers provide context for all data cells below them.                                                    |
| **[[2.4.7 Row Headers]]**                | Cover **vertical headers**: th elements in first column of each row, labeling rows, scope="row" for accessibility, and how row headers provide context for all data cells to their right.                                         |
| **[[2.4.8 When to Use Tables]]**         | Clarify **appropriate table usage**: truly tabular data (spreadsheets, schedules, comparisons), data with row-column relationships; distinguish from layout, lists, or card-based presentations.                                  |
| **[[2.4.9 When NOT to Use Tables]]**     | Address **table misuse**: never for page layout, not for single-column data (use lists), not for forms (use form elements), understanding why layout tables harm accessibility and responsiveness.                                |
| **[[2.4.10 Basic Table Styling]]**       | Introduce **CSS for tables**: border-collapse, cell padding, border styles, width control, text alignment; creating readable tables while maintaining semantic structure.                                                         |
