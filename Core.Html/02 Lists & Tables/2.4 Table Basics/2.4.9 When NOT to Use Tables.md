|---|---|---|---|
|**2.4.9 When NOT to Use Tables**|Address table misuse|Layout, Forms, Lists|Never use a table for visual positioning.|
|**[[2.4.9.1 The Layout Sin]]**|History|1990s|Do not use tables to put a sidebar next to the main content (use CSS Flexbox/Grid).|
|**[[2.4.9.2 Accessibility]]**|Harm|Confusion|Screen readers treat layout tables as data, confusing the user with "Row 1, Column 2".|
|**[[2.4.9.3 Responsiveness]]**|Rigidity|Scrollbars|Layout tables break on mobile screens; CSS layouts adapt.|
|**[[2.4.9.4 SEO]]**|Parsing|Semantics|Search engines want to see semantic tags (`<article>`, `<nav>`), not `<td>`.|
|**[[2.4.9.5 Forms]]**|Alignment|Labels/Inputs|Don't use tables to align form labels; use CSS grid or block elements.|
|**[[2.4.9.6 Performance]]**|Rendering|Reflow|Tables are computationally "heavy" to render compared to divs.|
