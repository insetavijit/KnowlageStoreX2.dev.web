# 2.4.2 Standard Filters

Liquid's power comes from its filters. Because you cannot perform arbitrary logic (like `variable + 1`) in standard output tags, you rely on Filters to perform almost all data manipulation.

Syntax: `{{ input | filter_name: argument }}`

---

## 2.4.2.1 String Filters

Liquid excels at text manipulation.

*   `{{ "hello" | capitalize }}` -> "Hello"
*   `{{ "HeLLo" | downcase }}` -> "hello"
*   `{{ "hello" | append: ".com" }}` -> "hello.com"
*   `{{ "<b>Hello</b>" | strip_html }}` -> "Hello" (Crucial for SEO descriptions)
*   `{{ "A long story..." | truncate: 10 }}` -> "A long..." (Cuts to 10 chars usually including ellipsis)

**Eleventy Specific**:
*   `slugify`: `{{ "My Title" | slugify }}` -> "my-title"

---

## 2.4.2.2 Array Filters

Working with collections (Arrays of Objects) is a daily task in Eleventy.

*   `map`: Extracts a specific property from a list of objects.
    *   `{{ posts | map: "title" }}` -> Returns array of titles.
*   `join`: Turns an array into a string.
    *   `{{ tags | join: ", " }}` -> "tech, design, writing"
*   `sort`: Reorders the array.
    *   `{{ pages | sort: "date" }}`
*   `where`: Filters an array by property (Crucial for avoiding `if` loops).
    *   `{% assign featured = posts | where: "featured", true %}`

---

## 2.4.2.3 Date Formatting

Liquid uses Ruby's `strftime` syntax for dates. It is robust but syntactically different from JavaScript libraries.

`{{ page.date | date: "%B %d, %Y" }}`
*   Output: "January 01, 2024"

Common Codes:
*   `%Y`: 2024 (Year)
*   `%m`: 01 (Month number)
*   `%B`: January (Full month name)
*   `%d`: 01 (Day)

---

## 2.4.2.4 Logic Filters (The Default)

The most important whitespace-saver in Liquid is `default`.

**Bad**:
```liquid
{% if user.name %}
  {{ user.name }}
{% else %}
  Guest
{% endif %}
```

**Good**:
```liquid
{{ user.name | default: "Guest" }}
```

This works for `null`, `undefined`, `false`, and empty strings.

---

## 2.4.2.5 Math Filters

Since Liquid prevents `{{ 1 + 1 }}`, you must use math filters. They chain left-to-right.

`{{ 10 | plus: 5 | divided_by: 3 }}`
*   (10 + 5) / 3 = 5

**Caution**: Liquid math handles Types strictly. `divided_by: 2` might return an integer (rounded down) if the input is an integer. To make it a float, divide by a float: `divided_by: 2.0`.
