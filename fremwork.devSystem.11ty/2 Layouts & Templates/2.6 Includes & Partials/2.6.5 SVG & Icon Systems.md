# 2.6.5 SVG & Icon Systems

Handling icons is a critical part of modern web development. While you *can* use `<img>` tags (`<img src="icon.svg">`), this prevents you from styling the icon (color, size) with CSS variables.

The Eleventy way is typically **Inline SVGs** using the Include system.

---

## 2.6.5.1 Inline SVG (The Default Strategy)

The simplest way to use an icon is to render the raw SVG code directly into your markup.

**`src/_includes/icons/star.svg`**:
```html
<svg viewBox="0 0 24 24" ...>
  <path d="..." />
</svg>
```

**Usage**:
```html
<button>
  Star 
  {% include "icons/star.svg" %}
</button>
```

**Why do this?**
Because the `<svg>` is now part of the DOM, you can target it with CSS:
```css
button:hover svg {
  fill: yellow;
}
```

---

## 2.6.5.2 Styling Inline SVGs (`currentColor`)

To make your icons strictly follow the text color of their container, edit your raw SVGs to remove hardcoded hex codes.

**Bad**:
`fill="#000000"`

**Good**:
`fill="currentColor"`

Now, your icon behaves like text. If you set `color: red` on the parent, the icon becomes red automatically.

---

## 2.6.5.3 Passing Classes (The Challenge)

Standard `{% include %}` essentially copy-pastes the file. You cannot easily inject a class *into* the `<svg>` tag found inside that file unless you use a Shortcode or a specific plugin.

**Workaround (Wrapper Strategy)**:
Wrap the include in a span.
```html
<span class="icon-large">
  {% include "icons/star.svg" %}
</span>
```

**Advanced Solution (Nunjucks Macro)**:
Define a macro `icon()` that accepts a name and class, loads the file using `fs`, manipulates the string, and outputs it. This usually requires custom JS (See **[[2.3.6 Nunjucks Environment]]**).

---

## 2.6.5.4 SVG Sprites (Performance)

If you have 50 icons on a page (e.g., a dashboard), inlining 50 chunks of SVG code bloats your HTML size.

**The Sprite Solution**:
1.  Create `_includes/sprite.svg` containing all your icons wrapped in `<symbol id="icon-name">`.
2.  Include `sprite.svg` heavily hidden at the top of your `<body>`.
3.  Use `<use>` tags throughout the page.

```html
<svg class="icon">
  <use href="#icon-star"></use>
</svg>
```

---

## 2.6.5.5 Accessibility (A11y)

Icons are confusing to screen readers.

1.  **Decorative Icons**: (e.g., star next to "Favorite"):
    Add `aria-hidden="true"` to the SVG or parent span.

2.  **Functional Icons**: (e.g., a search icon button with no text):
    The button needs an `aria-label`.
    `<button aria-label="Search">{% include "search.svg" %}</button>`

## Quick Reference

| Method | Pros | Cons |
| :--- | :--- | :--- |
| **`<img>`** | Cached by browser | No CSS styling control. |
| **Inline Include** | Full CSS control | Bloats HTML size if reused. |
| **Sprite** | Fast, Cached | Harder dev workflow. |
| **Font Icon** | Easy | Bad a11y, rendering quirks. |
