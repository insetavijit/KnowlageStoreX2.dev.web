# 2.7.1 WebC Setup

**WebC** is a serializer for standard Web Components. It allows you to write Single File Components (HTML + CSS + JS) that compile to standard HTML at build time. It is Eleventy's answer to React/Vue components, but without the client-side JavaScript cost.

It is an official plugin maintained by the Eleventy team.

---

## 2.7.1.1 Installation

First, install the plugin from npm:

```bash
npm install @11ty/eleventy-plugin-webc
```

Then, add it to your `.eleventy.js` config:

```javascript
const pluginWebc = require("@11ty/eleventy-plugin-webc");

module.exports = function(eleventyConfig) {
  eleventyConfig.addPlugin(pluginWebc, {
    // We will configure this in the next section
    components: "src/_components/**/*.webc" 
  });
};
```

---

## 2.7.1.2 Defining Components (Auto-Discovery)

The most powerful feature of WebC is its component lookup system. You provide a "glob pattern" (file path wildcard), and WebC effectively "imports" every file it finds there as a global component.

**Config**:
```javascript
components: "src/_components/**/*.webc"
```

**File System**:
```text
src/
└── _components/
    ├── my-footer.webc
    └── user-card.webc
```

---

## 2.7.1.3 The No-Import Magic

Because of the auto-discovery above, you do **not** need to import components at the top of your files.

**src/index.webc**:
```html
<!DOCTYPE html>
<html lang="en">
<body>
  <!-- This works automatically! -->
  <my-footer></my-footer>
  
  <user-card></user-card>
</body>
</html>
```

The tag name `<my-footer>` matches the filename `my-footer.webc`. If you change the filename, the tag name changes.

---

## 2.7.1.4 Naming Conventions (The Dash Rule)

WebC strictly follows the **HTML Specification for Custom Elements**. This means all component filenames (and thus tag names) **MUST contain a dash**.

*   ✅ `my-card.webc` -> `<my-card>`
*   ✅ `site-header.webc` -> `<site-header>`
*   ❌ `card.webc` -> `<card>` (Will be treated as a standard HTML span, not a component)

This prevents your components from colliding with future HTML tags (like `<dialog>` or `<details>`).

---

## 2.7.1.5 Tooling (VS Code)

Since `.webc` files are HTML files with super-powers, standard HTML highlighting works okay, but the official extension is better.

**Recommendation**: Install the **WebC** extension for VS Code. It provides:
1.  Syntax highlighting for `.webc` files.
2.  Intellisense for WebC-specific attributes (`@html`, `:prop`).
3.  Folding and formatting support.

Without it, treat `.webc` files as HTML in your editor settings.
