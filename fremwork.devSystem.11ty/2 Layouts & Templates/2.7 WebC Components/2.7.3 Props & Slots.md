# 2.7.3 Props & Slots

WebC follows the standard HTML component model:
1.  **Props** are passed via Attributes.
2.  **Content** is passed via Slots (Children).

---

## 2.7.3.1 Defining Props

In many frameworks (React, Vue), you must explicitly declare which props a component accepts. In WebC, **any attribute** placed on the component tag becomes a variable available in the template.

**Usage**:
```html
<my-card title="Hello World" icon="star"></my-card>
```

**Component (my-card.webc)**:
```html
<div class="card">
  <!-- 'icon' and 'title' are just variables now -->
  <img src="/icons/{{ icon }}.svg" alt="">
  <h2>{{ title }}</h2>
</div>
```

---

## 2.7.3.2 Dynamic Bindings (`:`)

To pass a variable (not a string literal) or a non-string type (like a number or boolean), prefix the attribute with a colon `:`.

**Usage**:
```html
<!-- 'post.title' is evaluated as a variable -->
<my-card :title="post.title" :is-featured="true"></my-card>
```

If you forget the colon (`title="post.title"`), the value of the variable inside the component will be the literal string `"post.title"`.

---

## 2.7.3.3 The Slot Mechanism

The `<slot>` element is a placeholder. It renders whatever "children" were passed to the component.

**Usage**:
```html
<my-button>
  Click <strong>Me</strong>
</my-button>
```

**Component (my-button.webc)**:
```html
<button class="btn">
  <!-- "Click <strong>Me</strong>" appears here -->
  <slot>Default Text</slot>
</button>
```

If no content is provided, the content *inside* the `<slot>` tags ("Default Text") is rendered as a fallback.

---

## 2.7.3.4 Named Slots

For complex components (like a Layout or a Modal), you often need multiple injection points.

**Usage**:
```html
<my-layout>
  <h1 slot="header">Page Title</h1>
  <p>Main Content</p>
  <footer slot="footer">Copyright</footer>
</my-layout>
```

**Component (my-layout.webc)**:
```html
<div class="page">
  <header>
    <slot name="header"></slot>
  </header>
  
  <main>
    <!-- Content without a slot name goes here (Default Slot) -->
    <slot></slot>
  </main>
  
  <slot name="footer"></slot>
</div>
```

---

## 2.7.3.5 Spreading Attributes (`@attributes`)

A common problem: You add `class="big"` to `<my-button>`, but it doesn't apply to the actual `<button>` inside. The outer `<my-button>` tag often vanishes (if `webc:nokeep` is set or inferred), dropping the class.

To fix this, you tell WebC where to put the "leftover" attributes.

**Usage**:
```html
<my-button class="big" id="submit-btn"></my-button>
```

**Component (my-button.webc)**:
```html
<button class="btn" @attributes>
  <slot></slot>
</button>
```

The `@attributes` keyword spreads all attributes (class, id, data-*) from the host component onto this specific element.

## Quick Reference

| Task | Syntax | Example |
| :--- | :--- | :--- |
| **Pass String** | `attr="val"` | `title="Hello"` |
| **Pass Variable** | `:attr="var"` | `:title="myVar"` |
| **Pass Content** | Child nodes | `<comp>Content</comp>` |
| **Named Content** | `slot="name"` | `<div slot="header">` |
| **Spread Attrs** | `@attributes` | `<button @attributes>` |
| **Private Prop** | `@prop` | Prevents attribute output. |
