# 3.1.4 Collection Items

When you loop through a collection (`{% for post in collections.post %}`), the variable `post` is a **Collection Item**.

Understanding the anatomy of this object is crucial because it is how you access everything about a page.

---

## 3.1.4.1 The Data Object (`.data`)

The most commonly used property is `item.data`. This object contains the **entire Data Cascade** for that file.

*   **Front Matter**: `item.data.title`, `item.data.tags`.
*   **Directory Data**: `item.data.layout`.
*   **Computed Data**: Any computed properties.

**Usage**:
```html
<h1>{{ post.data.title }}</h1>
<div class="author">By {{ post.data.author }}</div>
```

---

## 3.1.4.2 Metadata Properties

Directly on the item object (not inside `.data`), Eleventy exposes key metadata.

*   `item.date`: The JavaScript Date object for the post used for sorting.
*   `item.url`: The final permalink where this page will live (e.g., `/blog/my-post/`).
*   `item.fileSlug`: The filename without extension (e.g., `my-post`).

**Usage**:
```html
<a href="{{ post.url }}">Read More</a>
<time>{{ post.date | date: "%Y-%m-%d" }}</time>
```

---

## 3.1.4.3 Rendered Content (`.templateContent`)

If you want to display the full body of the post inside another page (aka "The Feed Pattern"), you use `item.templateContent`.

```html
{% for post in collections.post %}
  <article>
    <h1>{{ post.data.title }}</h1>
    <!-- Inherit styles for the content -->
    <div class="prose">
      {{ post.templateContent | safe }}
    </div>
  </article>
{% endfor %}
```

**Warning**: This contains the *compiled HTML* of the Markdown file. It assumes your CSS handles the styling.

---

## 3.1.4.4 File Paths

For debugging or advanced plugin logic, you often need to know where the file came from.

*   `item.inputPath`: `./src/posts/2023-01-01.md`
*   `item.outputPath`: `./_site/posts/2023-01-01/index.html`

Note that `templateContent` is not processed until it is requested, so accessing these properties is fast, but accessing `templateContent` triggers a render.

---

## 3.1.4.5 Navigation Helpers

Eleventy provides filter helpers to find adjacent items in a collection.

**Nunjucks/Liquid**:
```html
{% set nextPost = collections.post | getNextCollectionItem(page) %}
{% set prevPost = collections.post | getPreviousCollectionItem(page) %}

{% if nextPost %}
  <a href="{{ nextPost.url }}">Next: {{ nextPost.data.title }}</a>
{% endif %}
```

This logic automatically finds the current page in the `collections.post` array and returns the neighbor.
