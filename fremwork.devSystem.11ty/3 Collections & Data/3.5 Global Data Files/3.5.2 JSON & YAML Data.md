# JSON & YAML Data: Static Data Storage

JSON and YAML files in `_data` are the simplest way to store global data. They're pure data—no logic, no computation—just structured information that Eleventy parses and makes available everywhere.

---

## The Fundamental Principle: Data Without Logic

JSON and YAML files contain static data that never changes during the build. They're perfect for:

- Site metadata (name, URL, author)
- Navigation menus
- Footer links
- Social media profiles
- Configuration constants

Think of them as **configuration files** for your site.

---

## JSON Format

JSON is the most common format due to its simplicity and universal support:

**`_data/site.json`:**
```json
{
  "name": "My Awesome Blog",
  "url": "https://example.com",
  "author": {
    "name": "Jane Doe",
    "email": "jane@example.com"
  },
  "socialLinks": [
    { "platform": "twitter", "url": "https://twitter.com/jane" },
    { "platform": "github", "url": "https://github.com/jane" }
  ]
}
```

**Template Usage:**
```html
<title>{{ site.name }}</title>
<a href="{{ site.author.email }}">Contact</a>

{% for link in site.socialLinks %}
  <a href="{{ link.url }}">{{ link.platform }}</a>
{% endfor %}
```

---

## YAML Format

YAML is more human-readable, supports comments, and handles multiline strings elegantly:

**`_data/navigation.yaml`:**
```yaml
# Main site navigation
main:
  - label: "Home"
    url: "/"
  - label: "About"
    url: "/about/"
  - label: "Blog"
    url: "/blog/"

# Footer navigation
footer:
  - label: "Privacy Policy"
    url: "/privacy/"
  - label: "Terms of Service"
    url: "/terms/"
```

**Template Usage:**
```html
<nav>
  {% for item in navigation.main %}
    <a href="{{ item.url }}">{{ item.label }}</a>
  {% endfor %}
</nav>
```

---

## JSON vs YAML: When to Use Each

| Aspect | JSON | YAML |
| :--- | :--- | :--- |
| **Readability** | More verbose | More concise |
| **Comments** | Not supported | Supported |
| **Quotes** | Required for strings | Usually optional |
| **Multiline** | Escaped `\n` | Native support |
| **Familiarity** | Universal in web dev | Common in DevOps |

**Rule of Thumb:** Use JSON for simple data, YAML for complex structures with comments.

## Common Patterns

**Site Metadata:**
```json
{
  "title": "My Site",
  "description": "A site about things",
  "language": "en",
  "baseUrl": "https://example.com"
}
```

**Navigation Menu:**
```yaml
- label: "Home"
  url: "/"
  icon: "home"
- label: "Products"
  url: "/products/"
  children:
    - label: "Widget A"
      url: "/products/widget-a/"
```

## Common Pitfalls and Solutions

**Problem:** JSON parse error crashes the build.

**Cause:** Invalid JSON syntax (trailing commas, missing quotes).

**Solution:** Use a JSON validator. VS Code highlights errors automatically.

---

**Problem:** YAML indentation errors.

**Cause:** Mixed tabs and spaces, or incorrect nesting.

**Solution:** Use 2-space indentation consistently. Configure your editor.

---

**Problem:** Variable is `[object Object]` in template.

**Cause:** You're trying to print an object directly instead of its properties.

**Solution:** Access specific properties: `{{ site.name }}` not `{{ site }}`.
