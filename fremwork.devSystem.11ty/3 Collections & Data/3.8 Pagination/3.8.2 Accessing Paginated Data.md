# Accessing Paginated Data: Working with Page Chunks

Inside a paginated template, you need to access the **specific subset of items** assigned to the current page. Eleventy provides a `pagination` object with all the information you need.

---

## The Fundamental Principle: Each Page Gets Its Slice

When you paginate 100 items with `size: 10`, each generated page receives:
- Page 0: Items 0-9
- Page 1: Items 10-19
- Page 2: Items 20-29

The `pagination.items` array contains only that page's slice.

---

## The Pagination Object

Inside your template, `pagination` provides:

| Property | Type | Description |
| :--- | :--- | :--- |
| `items` | Array | The slice of data for this page |
| `pageNumber` | Number | Current page index (0-based) |
| `pages` | Array | Array of all page chunks |
| `hrefs` | Array | URLs for all pages |
| `href.first` | String | URL of first page |
| `href.last` | String | URL of last page |
| `href.previous` | String/null | URL of previous page |
| `href.next` | String/null | URL of next page |

---

## Iterating Over Items

The most common pattern:

```html
{% for post in pagination.items %}
  <article>
    <h2><a href="{{ post.url }}">{{ post.data.title }}</a></h2>
    <p>{{ post.data.excerpt }}</p>
  </article>
{% endfor %}
```

For collections, each item has:
- `item.data` — Front matter
- `item.url` — Permalink
- `item.date` — Date
- `item.templateContent` — Rendered body

For raw data (JSON arrays), the structure matches your source data.

---

## Page Information

Display context about the current page:

```html
<p>Showing {{ pagination.items.length }} posts</p>
<p>Page {{ pagination.pageNumber + 1 }} of {{ pagination.pages.length }}</p>
```

Note: `pageNumber` is 0-indexed, so add 1 for human-friendly display.

---

## Total Items Count

```html
{% set totalItems = 0 %}
{% for page in pagination.pages %}
  {% set totalItems = totalItems + page.length %}
{% endfor %}
<p>{{ totalItems }} total posts</p>
```

Or calculate from the original data source if available.

---

## Checking If Empty

```html
{% if pagination.items.length > 0 %}
  {% for post in pagination.items %}...{% endfor %}
{% else %}
  <p>No posts found.</p>
{% endif %}
```

---

## Common Pitfalls and Solutions

**Problem:** `pagination.items` is undefined.

**Cause:** Not inside a paginated template or missing `pagination:` front matter.

**Solution:** Verify the `pagination` block is properly configured.

---

**Problem:** Items have unexpected structure.

**Cause:** Paginating raw data vs collections have different shapes.

**Solution:** Log `{{ pagination.items[0] | dump }}` to inspect structure.

---

## Quick Reference

| Access | Value |
| :--- | :--- |
| **Current Items** | `pagination.items` |
| **Page Number** | `pagination.pageNumber` (0-indexed) |
| **Total Pages** | `pagination.pages.length` |
| **All URLs** | `pagination.hrefs` |
| **Item Properties** | `item.data.title`, `item.url` |
