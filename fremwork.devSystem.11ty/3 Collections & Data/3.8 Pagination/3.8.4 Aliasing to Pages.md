# Aliasing to Pages: Individual Detail Pages

When you set `size: 1`, each item in your data gets its own page. The **alias** property gives you a clean variable name to access that single item.

---

## The Fundamental Principle: Data â†’ Pages

One template + a data array = multiple pages. Each page represents one item.

```yaml
---
pagination:
  data: products
  size: 1
  alias: product
permalink: "products/{{ product.slug }}/"
---

<h1>{{ product.name }}</h1>
<p>{{ product.description }}</p>
<p>Price: {{ product.price }}</p>
```

From a single `products.json` with 100 products, this generates 100 product detail pages.

---

## Why Use Alias?

Without alias, you'd access `pagination.items[0]`:

```html
<!-- Clunky -->
<h1>{{ pagination.items[0].name }}</h1>
```

With alias:

```html
<!-- Clean -->
<h1>{{ product.name }}</h1>
```

---

## Basic Example

**`_data/team.json`:**
```json
[
  { "name": "Alice", "role": "Developer", "slug": "alice" },
  { "name": "Bob", "role": "Designer", "slug": "bob" },
  { "name": "Charlie", "role": "Manager", "slug": "charlie" }
]
```

**`team-member.njk`:**
```yaml
---
pagination:
  data: team
  size: 1
  alias: member
permalink: "team/{{ member.slug }}/"
layout: base.njk
---

<h1>{{ member.name }}</h1>
<p>Role: {{ member.role }}</p>
```

**Output:**
- `/team/alice/index.html`
- `/team/bob/index.html`
- `/team/charlie/index.html`

## With Collections

You can also alias collection items:

```yaml
---
pagination:
  data: collections.post
  size: 1
  alias: post
permalink: "articles/{{ post.data.slug }}/"
---

<article>
  <h1>{{ post.data.title }}</h1>
  {{ post.templateContent | safe }}
</article>
```

This creates an alternative URL structure for existing posts.

---

## Generating Tag/Category Pages

A powerful pattern for creating dedicated tag pages:

```yaml
---
pagination:
  data: collections
  size: 1
  alias: tag
  filter:
    - all
    - tagList
permalink: "tags/{{ tag | slugify }}/"
---

<h1>Posts tagged "{{ tag }}"</h1>
{% for post in collections[tag] %}
  <a href="{{ post.url }}">{{ post.data.title }}</a>
{% endfor %}
```

This generates one page per tag (e.g., `/tags/javascript/`, `/tags/css/`).

## Common Pitfalls and Solutions

**Problem:** Alias variable is undefined.

**Cause:** `alias` must match `size: 1`.

**Solution:** Ensure `size: 1` when using alias.

---

**Problem:** Permalink uses numbered pages instead of slug.

**Cause:** Dynamic permalink uses wrong variable.

**Solution:** Reference the alias variable: `{{ product.slug }}`.

---

## Quick Reference

| Config | Purpose |
| :--- | :--- |
| `size: 1` | One item per page |
| `alias: item` | Variable name for that item |
| `permalink` | Should use alias variable |
| `filter` | Exclude specific values |
