# Basic Pagination: Splitting Data Across Pages

Pagination is Eleventy's mechanism for creating **multiple output pages from a single template**. Instead of manually creating `/blog/page-1/`, `/blog/page-2/`, you define the pattern once and Eleventy generates all the pages.

---

## The Fundamental Principle: One Template, Many Pages

A paginated template is like a **cookie cutter**. You define the shape (template), and Eleventy stamps out copies for each chunk of data.

```yaml
---
pagination:
  data: collections.post
  size: 10
---
```

This single template generates:
- Page 1: Posts 1-10
- Page 2: Posts 11-20
- Page 3: Posts 21-30
- ... and so on

---

## Basic Syntax

```yaml
---
layout: base.njk
pagination:
  data: collections.post
  size: 10
permalink: "blog/{% if pagination.pageNumber > 0 %}page/{{ pagination.pageNumber + 1 }}/{% endif %}index.html"
---

<h1>Blog Archive</h1>
{% for post in pagination.items %}
  <article>
    <h2>{{ post.data.title }}</h2>
    <a href="{{ post.url }}">Read more</a>
  </article>
{% endfor %}
```

---

## Key Properties

| Property | Type | Description |
| :--- | :--- | :--- |
| `data` | String | Data path to paginate (collection or global data) |
| `size` | Number | Items per page |
| `alias` | String | Optional variable name for current item(s) |
| `reverse` | Boolean | Reverse the data before pagination |
| `filter` | Array | Values to exclude |

---

## Data Sources

You can paginate:

**Collections:**
```yaml
pagination:
  data: collections.post
```

**Global Data:**
```yaml
pagination:
  data: products   # From _data/products.json
```

**Front Matter Arrays:**
```yaml
items:
  - name: "Item 1"
  - name: "Item 2"
pagination:
  data: items
```

---

## Size Variations

**Standard Lists (size > 1):**
```yaml
size: 10  # 10 items per page
```

**Individual Pages (size = 1):**
```yaml
size: 1  # One page per item (see Aliasing)
```

---

## Output Files

By default, pagination creates numbered folders:
- `/blog/index.html` (page 0)
- `/blog/1/index.html` (page 1)
- `/blog/2/index.html` (page 2)

Customize with dynamic permalinks (covered in later sections).

---

## Common Pitfalls and Solutions

**Problem:** Only one page is generated.

**Cause:** `size` is larger than the dataset.

**Solution:** Check `size` value and data length.

---

**Problem:** Pages have wrong items.

**Cause:** Data source is modified elsewhere.

**Solution:** Verify the `data` path points to the correct collection.
